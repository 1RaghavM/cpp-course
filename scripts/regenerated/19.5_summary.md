A **void pointer** (`void*`) is a special pointer type that can point to an object of any data type. Unlike a typed pointer such as `int*` or `double*`, a `void*` discards all type information — it only stores a raw memory address. This makes it the closest thing C++ has to a truly generic pointer.

Because the compiler does not know what type a `void*` points to, you **cannot dereference it directly**. Attempting `*ptr` on a `void*` is a compile error. You also cannot perform pointer arithmetic on it, since the compiler has no idea how many bytes each element occupies. To use the pointed-to value, you must first cast the `void*` back to the correct typed pointer using `static_cast`.

```cpp
#include <iostream>

void printInt(void* ptr)
{
    int* intPtr{ static_cast<int*>(ptr) };
    std::cout << *intPtr << '\n';
}

int main()
{
    int value{ 42 };
    void* voidPtr{ &value };
    printInt(voidPtr);
    return 0;
}
```

Casting to the wrong type is **undefined behavior** — the compiler will not catch the mistake. If you store the address of a `double` but cast to `int*`, the program silently reads garbage. This is why `void*` is considered unsafe: it shifts all responsibility for type correctness onto the programmer.

In modern C++, `void*` is rarely the right tool. Templates and `std::variant` provide type-safe alternatives for generic code. The main places you still encounter `void*` are at the boundary with C libraries (e.g., `std::malloc` returns `void*`, callback APIs like `qsort` pass `void*` context parameters) and in low-level memory allocators.

Key rules to remember: any typed pointer converts to `void*` implicitly, but converting back always requires an explicit `static_cast`. A `void*` can be compared with `==` and `!=` and can be `nullptr`. Avoid `void*` whenever a template or typed pointer will do the job.
