## Passing and Returning Structs

Structs often contain multiple members, making them larger than fundamental types. How you pass and return them affects both correctness and performance.

### Passing structs by value

When you pass a struct by value, the entire struct is copied into the function parameter. For small structs this is fine, but for structs containing strings or many members the copy becomes expensive.

### Passing structs by const reference

The preferred approach is to pass structs by `const` lvalue reference. This avoids the copy while preventing the function from modifying the original. Use this as your default for any struct parameter that the function only needs to read:

```cpp
#include <iostream>
#include <string>

struct Employee {
    std::string name;
    int id;
};

void printEmployee(const Employee& e) {
    std::cout << e.name << " (#" << e.id << ")\n";
}

int main() {
    Employee emp{ "Kai", 42 };
    printEmployee(emp);
}
```

Pass by non-const reference only when the function needs to modify the caller's struct.

### Returning structs by value

Return structs by value. The compiler applies **named return value optimization (NRVO)** in most cases, constructing the return value directly in the caller's memory and eliminating the copy entirely. This means returning even a large struct by value is typically free.

Do not return structs by reference to a local variable — the local is destroyed when the function ends, leaving a dangling reference.

### Guidelines

- **Read-only parameter:** pass by `const&`.
- **Function needs to modify caller's struct:** pass by non-const `&`.
- **Returning a struct you built inside the function:** return by value (NRVO handles it).

These rules mirror what you learned for fundamental types and references in Chapter 12, but they matter more for structs because copies are more expensive.
