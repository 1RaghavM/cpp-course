A **recursive function** is a function that calls itself. Every recursive function needs two things: a **base case** that stops the recursion, and a **recursive case** that breaks the problem into a smaller subproblem and calls itself.

Here is a classic example — computing the factorial of a number:

```cpp
#include <iostream>

int factorial(int n) {
    if (n <= 1)        // base case
        return 1;
    return n * factorial(n - 1);  // recursive case
}

int main() {
    std::cout << factorial(5) << '\n'; // 120
    return 0;
}
```

When `factorial(5)` is called, the call stack grows with each recursive call: `factorial(5)` waits on `factorial(4)`, which waits on `factorial(3)`, and so on until `factorial(1)` hits the base case and returns `1`. Then each waiting call completes in reverse order, multiplying as it goes: `1 * 2 * 3 * 4 * 5 = 120`. Each call gets its own stack frame with its own copy of `n`.

**Stack overflow** is the main danger. Every recursive call adds a frame to the call stack, and the stack has a finite size (typically 1-8 MB). If the base case is missing or never reached, the function calls itself infinitely until the stack is exhausted and the program crashes. Always verify that every recursive path eventually reaches a base case and that the arguments move toward it on each call.

**When to use recursion vs. iteration:** Recursion is elegant for problems that are naturally self-similar — tree traversals, divide-and-conquer algorithms, and mathematical recurrences. However, a simple loop is usually faster and uses constant stack space. Use recursion when it makes the logic substantially clearer, and prefer iteration for straightforward counting or accumulation tasks.

A useful mental model: think of each recursive call as delegating a smaller version of the same job to a fresh copy of the function. Trust that the recursive call will return the correct answer for the smaller problem, then combine that answer to solve the current problem.
