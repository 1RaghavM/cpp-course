## C-style Arrays

Before `std::array` existed, C++ inherited **C-style arrays** from the C language. You will encounter them in legacy codebases, older textbooks, and low-level library interfaces, so understanding the basics is important even though `std::array` is preferred for new code.

A C-style array is declared by placing the array length in square brackets after the variable name:

```cpp
#include <iostream>

int main()
{
    int scores[4]{ 90, 85, 77, 92 }; // array of 4 ints
    scores[2] = 80;                   // modify element at index 2

    std::cout << "Length: " << sizeof(scores) / sizeof(scores[0]) << '\n';
    std::cout << "Element 0: " << scores[0] << '\n';
    std::cout << "Element 2: " << scores[2] << '\n';
}
```

The length must be a compile-time constant (a literal, `constexpr` variable, or `const` integral variable initialized with a constant). Unlike `std::array`, the compiler does **not** enforce bounds checking, so accessing an out-of-range index is undefined behavior with no runtime error.

### Initialization

You can brace-initialize a C-style array just like `std::array`. If you provide fewer initializers than the length, remaining elements are **value-initialized** (zero for fundamental types). Omitting the length entirely lets the compiler deduce it from the initializer count: `int vals[]{ 1, 2, 3 };` creates an array of length 3.

### `sizeof` on C-style Arrays

Applying `sizeof` to a C-style array returns the **total byte size** of the array (not the element count). To get the number of elements, divide by the size of a single element: `sizeof(arr) / sizeof(arr[0])`. In C++17 and later, prefer `std::size(arr)` from `<iterator>` for clarity and type safety.

### Why Prefer `std::array`

`std::array` wraps a C-style array but adds bounds-safe access via `.at()`, knows its own `.size()`, works naturally with range-based `for` loops, and cannot silently decay to a pointer. Use `std::array` for all new code and treat C-style arrays as a legacy feature you need to be able to read, not one you should choose by default.
