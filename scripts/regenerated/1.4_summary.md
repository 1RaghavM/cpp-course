# Variable Assignment and Initialization

In *Introduction to objects and variables*, you learned how to declare a variable with `int x;`. Declaring a variable allocates memory, but unless you explicitly provide a value, the variable is **uninitialized** — it holds whatever garbage was already in that memory. Using an uninitialized variable is one of the most common sources of bugs in C++.

This lesson covers how to give a variable a value, both at creation and afterward.

## Copy assignment

After a variable has been declared, you can give it a value using the **copy assignment operator** (`=`):

```cpp
int main()
{
    int x;      // declaration — x is uninitialized
    x = 10;     // copy assignment — x now holds 10
    x = 20;     // reassignment — x now holds 20

    return x;   // returns 20
}
```

Assignment replaces whatever value the variable previously held. You can assign to the same variable as many times as you need.

## Initialization

Rather than declaring first and assigning second, you can provide a value at the point of declaration. C++ offers several forms of **initialization**:

- **Copy initialization:** `int x = 5;` — inherited from C, familiar-looking, works fine for simple types.
- **Direct initialization:** `int x(5);` — uses parentheses. Behaves identically to copy initialization for integers, but the distinction matters later with more complex types.
- **Brace initialization (list initialization):** `int x{5};` — introduced in modern C++. This is the **preferred** form.

Brace initialization has one key advantage: it **disallows narrowing conversions**. If you accidentally try to store a value that doesn't fit — say, a fractional number in an `int` — brace initialization causes a compile error instead of silently dropping the fractional part. The other forms allow this quietly, hiding bugs.

## Initializing multiple variables

You can declare and initialize multiple variables of the same type on a single line:

```cpp
int a{1}, b{2}, c{3};
```

However, each variable needs its own initializer. Writing `int a, b{5};` declares `a` as uninitialized and only initializes `b`. For clarity, many programmers prefer one declaration per line, especially when each variable serves a distinct purpose.

## Default initialization and value initialization

Declaring `int x;` without any initializer leaves `x` uninitialized. Writing `int x{};` (empty braces) **value-initializes** it to `0`. Prefer `int x{0};` or `int x{};` over leaving variables uninitialized — it eliminates an entire class of bugs for no extra cost.
