## C-style Strings

Before `std::string` existed, C++ inherited **C-style strings** from C: null-terminated arrays of `char`. You already know C-style arrays from earlier lessons; a C-style string is simply a `char` array whose last meaningful element is followed by a **null terminator** (`'\0'`, value 0). The null terminator is how every C string function knows where the string ends.

```cpp
#include <cstring>
#include <iostream>

int main()
{
    char greeting[]{ "Hello" }; // compiler adds '\0', size is 6
    std::cout << std::strlen(greeting) << '\n'; // 5 (does not count '\0')

    char copy[10];
    std::strcpy(copy, greeting);       // copies chars + '\0' into copy
    std::cout << copy << '\n';         // Hello
    std::cout << std::strcmp(greeting, copy) << '\n'; // 0 (equal)
}
```

### The Null Terminator Matters

If you forget the null terminator, functions like `std::strlen`, `std::strcpy`, and `std::cout <<` will read past the end of the array into unrelated memory -- undefined behavior. When you initialize a `char` array with a string literal (`"Hello"`), the compiler appends `'\0'` automatically. When you fill the array yourself, you must place it manually.

### Common <cstring> Functions

- `std::strlen(s)` -- returns the number of characters before the null terminator (not counting it).
- `std::strcpy(dest, src)` -- copies `src` (including `'\0'`) into `dest`. You must ensure `dest` is large enough.
- `std::strcat(dest, src)` -- appends `src` onto the end of `dest`. The destination must have room for both strings plus the null terminator.
- `std::strcmp(a, b)` -- returns `0` if the strings are equal, a negative value if `a` comes before `b` lexicographically, and a positive value if `a` comes after `b`.

### Why Prefer std::string

C-style strings are error-prone: buffer overflows from `strcpy`/`strcat` into undersized arrays, off-by-one errors from forgetting the null terminator, and no built-in size tracking. `std::string` manages its own memory, knows its length, and supports comparison with `==` instead of `strcmp`. Use C-style strings only when interfacing with legacy C APIs or when you encounter them in existing code.
