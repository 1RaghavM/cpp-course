When you override a virtual function, the compiler silently creates a new function if the derived signature does not exactly match the base. A typo in the function name, a wrong parameter type, or a missing `const` means your function hides the base version instead of overriding it. Bugs like this compile cleanly and only surface at runtime when the wrong function is called through a base pointer.

The **`override`** specifier fixes this. Place it after the parameter list (and after `const`, if present) on any function that is meant to override a base virtual function. If the signatures do not match, the compiler emits an error immediately. Always use `override` on every derived override -- it costs nothing and catches real mistakes.

The **`final`** specifier prevents further overriding. A virtual function marked `final` cannot be overridden in any class that derives further. You can also mark an entire class `final` to prevent it from being used as a base class at all. Use `final` when a function's behavior must not change in deeper hierarchies.

```cpp
#include <iostream>

class Base {
public:
    virtual std::string identify() { return "Base"; }
};

class Derived : public Base {
public:
    std::string identify() override final { return "Derived"; }
};
```

Here `override` guarantees the signature matches `Base::identify()`, and `final` prevents any class derived from `Derived` from overriding `identify` again.

C++ also supports **covariant return types**. If a base virtual function returns `Base*` (or `Base&`), a derived override may return `Derived*` (or `Derived&`) instead, provided `Derived` is publicly derived from `Base`. The override is still valid, and callers through a base pointer receive a `Base*` while callers through a derived pointer receive a `Derived*`. This is useful for clone-style functions that return a pointer to the object's own type.

Key rules to remember: `override` and `final` are not keywords -- they are context-sensitive identifiers that only have special meaning after a function declaration. You can combine them as `override final` (in that order). A function does not need `virtual` in the derived class if the base already declared it virtual, but adding `override` makes the intent explicit.
