In lesson 19.1 you used `new` and `delete` to allocate single objects. C++ extends this to arrays with `new[]` and `delete[]`, letting you create arrays whose size is determined at runtime.

To allocate a dynamic array, write `new type[size]` where `size` can be any expression that evaluates to an integral value -- it does not need to be a compile-time constant. The result is a pointer to the first element. When you are done, call `delete[]` (not plain `delete`) on that pointer to free the entire array.

```cpp
#include <iostream>

int main()
{
    int count{};
    std::cin >> count;

    int* arr{ new int[count]{} }; // value-initialized to 0

    for (int i{ 0 }; i < count; ++i)
        std::cin >> arr[i];

    for (int i{ 0 }; i < count; ++i)
        std::cout << arr[i] << ' ';

    delete[] arr;
    return 0;
}
```

The `{}` after `new int[count]{}` value-initializes every element to zero. Without it, elements contain indeterminate values. You can also provide an initializer list: `new int[4]{ 1, 2, 3, 4 }`. Any elements not covered by the list are value-initialized.

A critical rule: **always match `new[]` with `delete[]`**. Using plain `delete` on an array allocated with `new[]` is undefined behavior -- it may leak memory or corrupt the heap. Likewise, using `delete[]` on a single-object allocation is also undefined.

Dynamic arrays have the same pointer-decay behavior as fixed arrays. You can index them with `[]`, iterate with pointer arithmetic, and pass them to functions that accept a pointer plus a size.

### Why prefer `std::vector`

Dynamic arrays via `new[]` require you to manually track the size and remember to call `delete[]`. If an exception is thrown between `new[]` and `delete[]`, the memory leaks. `std::vector` handles allocation, resizing, and cleanup automatically. In modern C++ you should almost always use `std::vector` instead of raw `new[]`. Understanding `new[]`/`delete[]` is still valuable because you will encounter it in legacy code and it deepens your understanding of how `std::vector` works internally.
