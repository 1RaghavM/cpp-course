## Class Template Argument Deduction (CTAD)

When you use a class template like `std::pair`, you normally specify template arguments explicitly: `std::pair<int, double> p{ 1, 3.14 };`. Starting with C++17, **class template argument deduction (CTAD)** lets the compiler deduce those arguments from the initializer, so you can write `std::pair p{ 1, 3.14 };` and the compiler figures out the types.

CTAD works with your own templates too. Given a struct template with a constructor (or, for aggregates in C++20, from the member types), the compiler matches the initializer types against the template parameters:

```cpp
#include <iostream>

template <typename T>
struct Wrapper
{
    T value;
};

int main()
{
    Wrapper w1{ 42 };       // CTAD: T = int
    Wrapper w2{ 3.14 };     // CTAD: T = double
    std::pair p{ 5, 2.0 };  // CTAD: pair<int, double>
    std::cout << w1.value << ' ' << w2.value << '\n';
}
```

For aggregate class templates (structs with no user-declared constructors), CTAD from brace initialization requires **C++20**. In C++17, CTAD only works when the class has a constructor whose parameters the compiler can use for deduction.

### Deduction Guides

Sometimes the compiler cannot deduce template arguments on its own -- for example, when you want a `Wrapper<std::string>` but pass a string literal (which is `const char*`). A **deduction guide** is a hint you write outside the class that tells the compiler how to map constructor argument types to template parameters:

```cpp
Wrapper(const char*) -> Wrapper<std::string>;
```

With this guide, `Wrapper w{"hello"};` deduces `T = std::string` instead of `T = const char*`. The standard library ships with many built-in deduction guides -- that is why `std::pair p{1, 2.0};` and `std::vector v{1, 2, 3};` work out of the box.

### When CTAD Does Not Apply

CTAD only triggers when **no** template arguments are provided. If you write even one argument explicitly (e.g., `std::pair<int> p{1, 2.0};`), CTAD is disabled and normal rules apply. CTAD also cannot deduce a type that does not appear in the initializer -- if a template parameter is used only internally, you must specify it explicitly.

### Best Practice

Use CTAD when the deduced types are obvious from context (e.g., `std::pair p{1, 2.0}`). Spell out the types explicitly when CTAD would deduce something surprising or when readability benefits from seeing the type.
