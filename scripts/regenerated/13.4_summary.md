## Converting an Enumeration to and from a String

Enumerators like `Color::red` are meaningful in code, but they cannot be printed directly as text. When you output an unscoped enumerator, the compiler converts it to its underlying integer — not the name you wrote. To display `"red"` instead of `0`, you need an explicit conversion function.

### Enum-to-String with a Switch

The standard approach is a function that maps each enumerator to a `std::string_view` using a `switch` statement:

```cpp
#include <iostream>
#include <string_view>
enum Color { red, green, blue };

constexpr std::string_view colorToString(Color c) {
    switch (c) {
        case red:   return "red";
        case green: return "green";
        case blue:  return "blue";
        default:    return "unknown";
    }
}
int main() {
    std::cout << colorToString(green) << '\n'; // prints: green
}
```

Returning `std::string_view` rather than `std::string` avoids a heap allocation on every call — the string literals already live in static storage. Marking the function `constexpr` lets the compiler resolve the conversion at compile time when the argument is a constant expression.

### String-to-Enum

The reverse direction takes a `std::string_view` and returns the matching enumerator. Because the input might not match any enumerator, a common pattern is to return a sentinel value (e.g. an `unknown` enumerator) or use `std::optional` (covered in 12.15). A simple approach with a sentinel:

Write a function that compares the input against each known string using `if`/`else if` and returns the corresponding enumerator, or returns a designated "invalid" enumerator when nothing matches.

### Why `constexpr` Matters Here

Because the switch body contains only `return` statements with literal values, the compiler can evaluate the entire function at compile time when called with a `constexpr` argument. This is free performance — no runtime cost, no branch mispredictions — and it works today with every major compiler under C++17 or later.

### Keep Them in Sync

The biggest maintenance risk is adding a new enumerator but forgetting to update the conversion functions. A practical guard: compile with `-Wall -Wswitch` (enabled by `-Wall`). The compiler will warn if a `switch` on an enum type is missing a case, catching the mismatch before it becomes a bug.
