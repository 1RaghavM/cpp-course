C++ lets you redefine the meaning of most operators for your own types. This is called **operator overloading** -- you write a function whose name is `operator` followed by the symbol (e.g. `operator+`), and the compiler calls that function whenever the operator is used with matching operand types.

Operator overloading does not let you invent new operators, change the number of operands an operator takes (its arity), or change the precedence or associativity of an operator. You can only change what an existing operator *does* when at least one operand is a user-defined type. Built-in operator meanings for fundamental types (`int`, `double`, etc.) cannot be changed.

Not every operator is overloadable. The ones you **cannot** overload include `::` (scope resolution), `sizeof`, `?:` (ternary), `.` (member access), and `.*` (member pointer access). Nearly everything else -- arithmetic, comparison, subscript, assignment, stream insertion/extraction -- is fair game.

There are three ways to implement an overloaded operator:

- **As a member function** of the class. The left operand becomes `*this`, so a binary operator takes only one explicit parameter. Best for operators that modify the left operand (like `+=`) or that need access to private members.
- **As a friend function** declared inside the class. This gives the function access to private members while keeping both operands as explicit parameters. Useful for symmetric operators where the left operand is not necessarily your class type.
- **As a normal (non-friend, non-member) function.** This works when the implementation only needs the class's public interface. It provides the loosest coupling.

```cpp
#include <iostream>

class Cents {
    int m_cents{};
public:
    Cents(int cents) : m_cents{cents} {}
    int getCents() const { return m_cents; }
    friend Cents operator+(const Cents& a, const Cents& b);
};

Cents operator+(const Cents& a, const Cents& b) {
    return Cents{a.m_cents + b.m_cents};
}

int main() {
    Cents sum{Cents{5} + Cents{9}};
    std::cout << sum.getCents() << '\n'; // 14
}
```

When choosing an approach, prefer the one that gives the operator the least access it needs. If public getters are sufficient, use a normal function. If private access is required and the operator is symmetric, prefer a friend. Reserve the member function form for operators that naturally modify the left operand or where the language requires it (like `=`, `[]`, `()`, and `->`).
