Programs model real-world systems, and real-world systems are built from objects that relate to each other. C++ recognizes four categories of object relationships, distinguished by two questions: **who owns the lifetime** of the related object, and **can the related object exist independently**?

**Composition** is the strongest relationship. A composed object (the "part") is created and destroyed by its owner (the "whole"). The part cannot exist on its own. A `Heart` inside a `Body` is composition -- when the body is destroyed, the heart goes with it. In code, this typically means the part is a direct member variable or is allocated in the constructor and freed in the destructor.

**Aggregation** is a weaker "has-a" relationship. The whole contains a reference or pointer to the part, but the part can exist before the whole is created and can survive after the whole is destroyed. A `Department` has `Professor` objects, but professors exist independently of any single department.

**Association** is a "uses-a" relationship where neither object owns the other. Both objects exist independently, and the link can be unidirectional or bidirectional. A `Doctor` and a `Patient` are associated -- each may reference the other, but neither manages the other's lifetime.

**Dependency** is the weakest relationship. One object temporarily uses another, typically as a function parameter, and holds no long-term reference to it. A function that takes a `std::ostream&` to print output depends on that stream only for the duration of the call.

```cpp
#include <iostream>

class Engine {
    int m_hp;
public:
    Engine(int hp) : m_hp{hp} {}
    int getHP() const { return m_hp; }
};

class Car {                    // composition: Car owns the Engine
    Engine m_engine;
public:
    Car(int hp) : m_engine{hp} {}
    void print(std::ostream& out) const { out << m_engine.getHP() << "hp\n"; }
};
```

In the example above, `Car` **composes** an `Engine` (direct member, same lifetime). The `print` function has a **dependency** on `std::ostream` -- it uses the stream only during the call and stores no reference to it. Recognizing which relationship applies guides you toward the right member type: value for composition, pointer or reference for aggregation and association, and function parameter for dependency.
