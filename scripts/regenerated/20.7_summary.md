In lesson 20.6 you wrote lambdas with empty capture clauses `[]`. That limits a lambda to using only its own parameters and local variables. **Captures** let a lambda access variables from the enclosing scope.

## Capture by value and by reference

Put a variable name inside the brackets to capture it **by value** (a copy is made when the lambda is defined). Prefix with `&` to capture **by reference** (the lambda sees the original variable).

```cpp
#include <iostream>

int main() {
    int offset{ 10 };
    int multiplier{ 3 };

    auto transform{ [offset, &multiplier](int x) {
        multiplier = 4;           // modifies the outer variable
        return x * multiplier + offset;
    } };

    std::cout << transform(5) << '\n'; // 5*4 + 10 = 30
    std::cout << multiplier << '\n';   // 4 (changed by lambda)
}
```

`offset` is captured by value, so the lambda works with a snapshot. `multiplier` is captured by reference, so assigning to it inside the lambda changes the original.

## Default captures

Writing every variable name gets tedious. A **default capture** tells the compiler how to capture anything the lambda body uses: `[=]` captures everything by value, `[&]` captures everything by reference. You can mix a default with specific overrides: `[=, &multiplier]` captures most things by value but `multiplier` by reference.

## Mutable lambdas

A value-captured variable is `const` inside the lambda by default. If you need to modify your local copy (without affecting the original), mark the lambda `mutable`:

```cpp
auto counter{ [count = 0]() mutable { return ++count; } };
```

Each call increments the lambda's internal copy of `count`, but no outer variable is touched.

## Guidelines

Prefer capturing specific variables over default captures so readers immediately see what the lambda depends on. Use capture by value unless the lambda genuinely needs to modify or observe changes to the outer variable.
