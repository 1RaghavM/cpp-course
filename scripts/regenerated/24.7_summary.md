When a derived class inherits from a base class, it gets all of the base class's public and protected member functions. But sometimes the derived class needs a function to behave differently. In C++, a derived class can **redefine** (hide) a base class member function by declaring a function with the same name and parameters in the derived class.

```cpp
#include <iostream>

class Base
{
public:
    void identify() { std::cout << "Base\n"; }
};

class Derived : public Base
{
public:
    void identify() { std::cout << "Derived\n"; }
};
```

When you call `identify()` on a `Derived` object, the compiler finds `Derived::identify()` first and calls it. The base version is **hidden**, not deleted -- you can still reach it explicitly with `Base::identify()`. This is compile-time **name hiding**, not runtime polymorphism (virtual function overriding comes in Chapter 25).

A common pattern is for the derived version to extend rather than fully replace the base behavior. The derived function calls the base version to do the shared work, then adds its own logic:

```cpp
void Derived::identify()
{
    Base::identify();              // reuse base logic
    std::cout << "...Derived\n";   // add derived-specific behavior
}
```

You must use the `Base::functionName()` syntax when calling the base version from inside the derived class. Writing just `identify()` inside `Derived::identify()` would call itself, causing infinite recursion.

Name hiding has an important scope rule: when the derived class redefines **any** overload of a function name, **all** base class overloads of that name become hidden. If `Base` has `print(int)` and `print(double)`, and `Derived` redefines only `print(int)`, then calling `print(3.14)` on a `Derived` object will call `Derived::print(int)` (with a narrowing conversion) rather than `Base::print(double)`. The base overloads can still be accessed via `Base::print(3.14)`.

Key takeaways: redefinition is resolved at compile time based on the static type of the object; it hides rather than overrides; and you call hidden base functions with the fully qualified `Base::` prefix.
