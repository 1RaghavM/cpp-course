## iostream: cout, cin, and endl

The `<iostream>` header gives your program two objects it needs constantly: `std::cout` for output and `std::cin` for input. These aren't functions — they're objects with overloaded operators, a concept you'll understand more deeply later.

### Sending output with std::cout

`std::cout` uses the insertion operator `<<` to push values to the terminal. You can chain multiple `<<` calls on one line:

```cpp
#include <iostream>

int main() {
    int score{ 42 };          // direct initialization (see: Variable assignment and initialization)
    std::cout << "Score: " << score << '\n';
    std::cout << "Double: " << score * 2 << '\n';
    return 0;
}
```

The `'\n'` character ends the line. You'll also see `std::endl`, which does the same thing but additionally flushes the output buffer — forcing the OS to write pending output immediately. Prefer `'\n'` in tight loops because flushing is slow; use `std::endl` when you need the output to appear before a blocking operation like `std::cin`.

### Reading input with std::cin

`std::cin` uses the extraction operator `>>` to read a value from the user and store it in a variable. The variable must already exist (see: Introduction to objects and variables):

```cpp
int age{};
std::cin >> age;
```

`std::cin` skips leading whitespace and stops at the first whitespace after the value, so reading two integers in sequence works naturally:

```cpp
int x{}, y{};
std::cin >> x >> y;   // reads two space- or newline-separated integers
```

In Python you call `input()` and cast manually. In C++ the type of the variable tells `>>` how to parse — an `int` variable causes it to parse an integer, a `double` causes it to parse a floating-point number. Type mismatches put `std::cin` into a fail state; you'll learn error handling later.

### The std:: prefix

Every name in the standard library lives in the `std` namespace. The prefix `std::` tells the compiler where to find `cout`, `cin`, and `endl`. Lesson 1-2 (Comments) mentioned that clear names reduce confusion — `std::cout` is unambiguous in a way that a bare `cout` (after `using namespace std;`) is not. Avoid `using namespace std;` in files you'll maintain.

### Practical mental model

- `std::cout << x` — push `x` toward the screen
- `std::cin >> x` — pull a value from the keyboard into `x`
- The arrows point in the direction data moves
