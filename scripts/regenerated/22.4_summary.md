When you have a named object (an l-value) and you want to invoke a move constructor or move assignment operator instead of a copy, you need a way to treat that l-value as an r-value reference. That is exactly what `std::move` does -- it is a cast, defined in `<utility>`, that converts its argument to an r-value reference. It does not move anything by itself.

Consider a simple `DynArray` class with a move constructor. If you write `DynArray b{ a };` the compiler sees `a` as an l-value and copies. Writing `DynArray b{ std::move(a) };` casts `a` to an r-value reference, so the move constructor runs instead and steals the internal resources.

```cpp
#include <iostream>
#include <utility>

class DynArray {
    int* m_data{};
    int m_size{};
public:
    DynArray(int sz) : m_data{new int[sz]{}}, m_size{sz} {}
    DynArray(DynArray&& src) noexcept
        : m_data{src.m_data}, m_size{src.m_size}
    { src.m_data = nullptr; src.m_size = 0; }
    ~DynArray() { delete[] m_data; }
    int size() const { return m_size; }
};
```

After `std::move(a)` and the move constructor runs, `a` is in a **moved-from state**: its pointer is `nullptr` and its size is `0`. The object is still valid and destructible, but you should not rely on its contents. The only safe operations are reassignment and destruction.

Key rules for using `std::move`:

- **Use it when you are done with an object** and want to transfer its resources elsewhere, such as pushing a local object into a container or returning from a function where copy elision does not apply.
- **Do not use a moved-from object** for anything other than reassignment or destruction.
- **Moving a `const` object silently falls back to copying** because a move constructor takes a non-const r-value reference, which cannot bind to a `const` r-value. The compiler quietly selects the copy constructor instead -- no error, no warning, just a performance surprise.
- **Do not `std::move` a return value** from a function -- this disables Return Value Optimization (RVO) and makes performance worse.
