## Multidimensional C-style Arrays

A C-style array can have more than one dimension. A **two-dimensional array** is declared by providing two sizes in square brackets: `int grid[3][4];` creates 3 rows of 4 columns (12 `int` elements total). You can extend this to three or more dimensions, but 2D is by far the most common.

### Declaration and Initialization

You can brace-initialize a 2D array with nested braces -- one inner pair per row:

```cpp
#include <iostream>

int main()
{
    int matrix[2][3]{
        {1, 2, 3},   // row 0
        {4, 5, 6}    // row 1
    };

    for (int r{ 0 }; r < 2; ++r)
    {
        for (int c{ 0 }; c < 3; ++c)
            std::cout << matrix[r][c] << ' ';
        std::cout << '\n';
    }
}
```

If you omit inner braces, elements fill in row-major order (row 0 first, then row 1, etc.). Missing initializers are zero-initialized, so `int m[3][4]{};` gives you a 3x4 grid of zeros.

### Memory Layout

C++ stores multidimensional arrays in **row-major order**: the elements of row 0 are contiguous in memory, followed by all of row 1, and so on. This means iterating over columns in the inner loop is cache-friendly, while iterating over rows in the inner loop is not. For performance-critical code, always make the rightmost index vary fastest.

### Traversal with Nested Loops

The standard pattern is two nested `for` loops: the outer loop iterates over rows and the inner loop over columns. Use `std::size_t` or a plain `int` for the loop counters. You can read, write, or accumulate values using `array[row][col]`.

### Limitations

Because these are C-style arrays, all the same caveats from earlier lessons apply: they decay to pointers when passed to functions (losing the size of the first dimension), they have no `.size()` member, and bounds are never checked. For fixed-size multidimensional data in modern C++, prefer a nested `std::array` when you have access to it. However, understanding 2D C-style arrays is essential for reading legacy code and competitive programming.
