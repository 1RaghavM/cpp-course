## Struct Miscellany

This lesson covers several practical struct topics that round out your working knowledge: nesting structs inside other structs, combining structs with enumerations, and understanding how much memory a struct actually occupies.

### Nested Structs

A struct member can itself be a struct. This lets you compose complex types from simpler building blocks without flattening everything into one level:

```cpp
#include <iostream>
#include <string>

struct Address {
    std::string city;
    int zip;
};

struct Employee {
    std::string name;
    Address addr;
};

int main() {
    Employee e{ "Alice", { "Denver", 80202 } };
    std::cout << e.name << " lives in "
              << e.addr.city << '\n';
}
```

When using aggregate initialization, nested members are initialized with their own set of braces. Access uses chained dot operators: `e.addr.city`.

### Structs with Enums

Enumerations and structs complement each other naturally. An enum defines a fixed set of states; a struct bundles that state with related data. For example, pairing a `Status` enum with a `Task` struct gives both type safety on the status field and a clean grouping of all task information.

### Struct Size and the `sizeof` Operator

You can query the size of a struct in bytes with `sizeof(StructName)` or `sizeof(object)`. The result is always a compile-time constant.

A struct's size is **at least** the sum of its members' sizes, but is often **larger** because of **padding**. The compiler inserts padding bytes between or after members so that each member sits at a memory address aligned to its type's alignment requirement (typically equal to the type's size). After the last member, the compiler may add **trailing padding** so the total struct size is a multiple of its largest member's alignment.

### Minimizing Padding

You can reduce wasted space by ordering members from largest alignment requirement to smallest. For example, placing a `double` (8-byte alignment) before a `char` (1-byte alignment) wastes less space than the reverse order. However, only optimize layout when you have measured a real memory concern -- readability comes first.

### Key Takeaways

- Nest structs freely to model hierarchical data; initialize nested members with inner braces.
- Pair enums with structs to get type-safe status or category fields.
- `sizeof` on a struct returns the total bytes including padding -- it may be more than the sum of member sizes.
- Member ordering affects padding and therefore struct size, but prefer logical grouping unless memory pressure demands otherwise.
