## Class Templates

You already know that **function templates** let you write a single function definition that works with multiple types. **Class templates** apply the same idea to structs (and classes): you define the struct once with placeholder type parameters, and the compiler generates concrete versions for each set of types you use.

### Defining a Class Template

Prefix the struct definition with `template<typename T>` (or multiple parameters like `template<typename T, typename U>`). Inside the struct, use the type parameters wherever a concrete type would normally go:

```cpp
#include <iostream>

template<typename T, typename U>
struct Pair
{
    T first{};
    U second{};
};

int main()
{
    Pair<int, double> p1{ 5, 3.14 };
    Pair<std::string, int> p2{ "age", 30 };
    std::cout << p1.first << ' ' << p1.second << '\n';
    std::cout << p2.first << ' ' << p2.second << '\n';
}
```

When the compiler sees `Pair<int, double>`, it generates a concrete struct with `int first` and `double second`. This process is called **template instantiation**, and each distinct combination of type arguments produces a separate type.

### Explicit Template Arguments Are Required (For Now)

At this point in the curriculum you must always supply the types inside angle brackets: `Pair<int, double>`. The compiler cannot yet deduce them from the initializer values. (Class template argument deduction, or CTAD, is a later topic.)

### Single Type Parameter Templates

When every member uses the same type, a single parameter suffices:

```cpp
template<typename T>
struct Triplet
{
    T a{};
    T b{};
    T c{};
};
```

You instantiate it the same way: `Triplet<int> t{ 1, 2, 3 };`.

### Using Class Templates with Functions

You can write function templates that accept class template instantiations. The function's own template parameters stand in for the class template's type arguments:

```cpp
template<typename T, typename U>
void printPair(const Pair<T, U>& p)
{
    std::cout << p.first << ' ' << p.second;
}
```

Here the compiler deduces `T` and `U` from the `Pair` you pass in -- function template argument deduction still works normally.

### Key Points

- Class templates are defined in header files because the compiler needs the full definition at every point of instantiation.
- Each unique set of template arguments creates a completely independent type: `Pair<int, int>` and `Pair<double, double>` are unrelated types.
- Use class templates whenever you find yourself duplicating a struct definition just to change the member types.
