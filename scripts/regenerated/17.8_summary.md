## C-style Array Decay

When you use a C-style array in most expressions, the compiler silently converts it into a pointer to its first element. This implicit conversion is called **array decay** (or "array-to-pointer decay"), and it is the single biggest reason C-style arrays are error-prone.

### How Decay Works

A C-style array has a type that encodes its length -- `int arr[5]` has type `int[5]`. But the moment you pass `arr` to a function or assign it to a pointer, the compiler drops the length and gives you a plain `int*`:

```cpp
#include <iostream>

void printSize(int* ptr, int length)
{
    // sizeof(ptr) is the size of a pointer, NOT the array
    std::cout << "pointer size: " << sizeof(ptr) << '\n';
    std::cout << "length: " << length << '\n';
}

int main()
{
    int arr[5] { 10, 20, 30, 40, 50 };
    std::cout << "array size: " << sizeof(arr) << '\n';
    printSize(arr, 5); // arr decays to int*
}
```

Inside `main`, `sizeof(arr)` reports the true byte size of the entire array (typically 20 on systems where `int` is 4 bytes). But after decay, `sizeof(ptr)` reports the size of a pointer (typically 8 bytes on a 64-bit system) -- the array length is gone.

### Why This Matters

Because the length is lost, every function that receives a decayed array must also receive the length as a separate parameter. Forgetting to pass the length, or passing the wrong value, causes out-of-bounds access -- a bug the compiler cannot catch. This is the root cause of countless buffer-overflow vulnerabilities in C and C++ code.

When you write a function parameter as `int arr[]`, the compiler treats it identically to `int* arr`. The bracket syntax is cosmetic; no length checking occurs.

### Prefer `std::array`

`std::array` does not decay. It carries its length as part of the type, so `sizeof` always works and functions receive the full type. This is a key reason modern C++ favors `std::array` (or `std::vector`) over C-style arrays. When you encounter legacy code that uses C-style arrays, watch for decay -- especially at function boundaries -- and pass the size explicitly.
