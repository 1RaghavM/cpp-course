When you instantiate a derived class, C++ does not simply construct the derived object in one step. Instead, construction happens in a well-defined order: **the base class portion is constructed first, then the derived class portion**. This makes intuitive sense — a derived class builds *on top of* the base, so the foundation must exist before the extension.

You can observe this directly by placing print statements inside constructors:

```cpp
#include <iostream>
struct Base
{
    Base()  { std::cout << "Base constructed\n"; }
    ~Base() { std::cout << "Base destroyed\n"; }
};
struct Derived : public Base
{
    Derived()  { std::cout << "Derived constructed\n"; }
    ~Derived() { std::cout << "Derived destroyed\n"; }
};
int main()
{
    Derived d;
    return 0;
}
```

Running this prints `Base constructed`, then `Derived constructed`. When `d` goes out of scope the output reverses: `Derived destroyed`, then `Base destroyed`. This reverse order of destruction mirrors how C++ destroys member variables — last constructed, first destroyed.

Why does the order matter? Consider a derived class whose constructor uses a member inherited from the base. If the base were not yet constructed, that member would be in an indeterminate state and any access would be undefined behavior. By guaranteeing base-first construction, C++ ensures every inherited member is fully initialized before the derived constructor body runs.

Destruction follows the opposite order for the same reason. The derived portion may reference base members during cleanup, so the derived destructor runs while the base is still intact. Only after the derived part is fully torn down does the base destructor execute.

This ordering is not something you opt into — it is automatic and enforced by the language. Even if you do not write explicit constructors or destructors, the compiler-generated defaults follow the same sequence. Understanding this order is essential groundwork for the next lessons, where you will learn how to pass arguments from a derived constructor to its base constructor.
