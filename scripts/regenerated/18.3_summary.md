## Introduction to Standard Library Algorithms

The `<algorithm>` and `<numeric>` headers provide dozens of ready-made functions that operate on iterator ranges. Instead of writing manual loops, you pass a pair of iterators (begin, end) and the library does the rest. This produces shorter, less error-prone code that clearly communicates intent.

```cpp
#include <algorithm>
#include <array>
#include <iostream>
#include <numeric>

int main()
{
    std::array data{ 7, 2, 9, 4, 1 };
    std::sort(data.begin(), data.end());
    auto it{ std::find(data.begin(), data.end(), 4) };
    int total{ std::accumulate(data.begin(), data.end(), 0) };

    std::cout << "Found 4 at index " << (it - data.begin()) << '\n';
    std::cout << "Sum: " << total << '\n';
}
```

### Commonly Used Algorithms

**`std::sort(begin, end)`** sorts the range in ascending order. For descending order you can pass a custom comparator function as a third argument.

**`std::find(begin, end, value)`** performs a linear search and returns an iterator to the first match, or `end` if the value is not found. Always compare the result against `end` before dereferencing.

**`std::count(begin, end, value)`** returns the number of elements equal to `value`.

**`std::min_element(begin, end)`** and **`std::max_element(begin, end)`** return iterators to the smallest and largest elements respectively. Dereference the returned iterator to get the actual value.

**`std::for_each(begin, end, func)`** calls `func` on every element in the range. You pass a named function (or function pointer) as the third argument.

**`std::accumulate(begin, end, init)`** lives in `<numeric>`, not `<algorithm>`. It returns the sum of `init` plus every element in the range. The type of `init` determines the return type, so use `0` for `int` sums and `0.0` for `double` sums.

### Why Prefer Algorithms Over Raw Loops

Algorithms eliminate common loop bugs (off-by-one errors, wrong comparison operators). They also convey meaning at a glance -- `std::sort` is immediately clear, while a hand-written sort loop requires the reader to verify correctness. As a guideline, reach for a standard algorithm before writing a manual loop.
