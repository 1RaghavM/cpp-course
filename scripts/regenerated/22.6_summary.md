A `std::unique_ptr` enforces single ownership -- exactly one pointer owns the resource at any time. But sometimes a resource legitimately has **multiple owners**. For example, several data structures might hold a pointer to the same configuration object, and the object should only be destroyed when the last owner is done with it. This is the problem `std::shared_ptr` solves.

`std::shared_ptr` uses **reference counting**. Internally it maintains a control block that tracks how many `shared_ptr` instances point to the same object. Every time you copy a `shared_ptr`, the count increments. Every time a `shared_ptr` is destroyed or reassigned, the count decrements. When the count reaches zero, the managed object is deleted. You can inspect the current count with `use_count()`.

Create a `shared_ptr` with `std::make_shared<T>(args...)`, which allocates the object and the control block in a single memory allocation (more efficient than using `new` directly):

```cpp
#include <iostream>
#include <memory>

int main()
{
    auto ptr1{ std::make_shared<int>(42) };
    std::cout << ptr1.use_count() << '\n'; // 1

    auto ptr2{ ptr1 }; // copy: count goes to 2
    std::cout << ptr1.use_count() << '\n'; // 2

    ptr2.reset(); // ptr2 releases ownership: count back to 1
    std::cout << ptr1.use_count() << '\n'; // 1
}
```

Key differences from `std::unique_ptr`: shared pointers **can be copied** (unique pointers cannot), shared pointers carry the overhead of the control block and atomic reference count operations, and shared pointers use `std::make_shared` instead of `std::make_unique`. Like `unique_ptr`, `shared_ptr` overloads `operator*` and `operator->` so you can use it like a raw pointer, and a default-constructed `shared_ptr` is null.

Prefer `std::unique_ptr` when single ownership is sufficient -- it is cheaper and expresses intent more clearly. Reach for `std::shared_ptr` only when you genuinely need shared ownership. Avoid creating a `shared_ptr` from a raw pointer that is already managed by another `shared_ptr`; always copy from an existing `shared_ptr` or use `make_shared` to prevent double-deletion.
