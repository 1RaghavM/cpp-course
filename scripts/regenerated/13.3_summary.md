## Unscoped Enumerator Integral Conversions

Unscoped enumerators implicitly convert to integral values. When you use an unscoped enumerator where an integer is expected, the compiler silently promotes it. This is called an **implicit integral conversion**, and it happens automatically — no cast required.

By default, the first enumerator has value `0`, the next `1`, and so on (unless you assign explicit values). Because of implicit conversion, you can pass an enumerator directly to `std::cout`, use it in arithmetic, or compare it to an integer:

```cpp
#include <iostream>

enum Color { red, green, blue };

int main()
{
    Color c{ green };
    std::cout << c << '\n';        // prints 1
    std::cout << c + 10 << '\n';   // prints 11
    if (c == 1)
        std::cout << "green\n";
    return 0;
}
```

The output is `1`, then `11`, then `green`. The enumerator `green` converts to `1` wherever an integer is needed.

### Converting Integers Back to Enumerators

The reverse direction does **not** happen implicitly. You cannot assign an integer to an enum variable without an explicit cast. Use `static_cast` to convert an integer to an enumerator:

```cpp
Color c2{ static_cast<Color>(2) }; // c2 is blue
```

This is the same `static_cast` you learned in lesson 4.12. The compiler trusts you that the integer corresponds to a valid enumerator — if you pass a value outside the defined range, the behavior is implementation-defined. Always validate user input before casting to an enum.

### Why This Matters

- **Reading input:** `std::cin` cannot read directly into an unscoped enum. You read into an `int`, validate it, then `static_cast` to the enum type.
- **Indexing and arithmetic:** Enumerators that implicitly convert to integers can be used as array indices or in math expressions without casts.
- **Comparisons:** You can compare enumerators to integers directly, but comparing two different enum types is a compiler error.

### Pitfalls

- Implicit conversion can hide bugs when you accidentally use an enumerator where an integer was intended. Scoped enums (covered in a later lesson) prevent this.
- Casting an out-of-range integer to an enum does not produce a compiler error — it silently gives you an invalid enumerator.
