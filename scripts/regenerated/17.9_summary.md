## Pointer Arithmetic and Subscripting

When a C-style array decays to a pointer (lesson 17.7), that pointer holds the address of element 0. **Pointer arithmetic** lets you move to other elements by adding or subtracting integers. The compiler scales the offset by the element size automatically, so `ptr + 2` means "two *elements* forward", not two bytes.

```cpp
#include <iostream>

int main()
{
    int arr[]{ 10, 20, 30, 40, 50 };
    int* ptr{ arr };          // points to arr[0]

    std::cout << *(ptr + 0) << '\n'; // 10
    std::cout << *(ptr + 3) << '\n'; // 40
    std::cout << *(ptr + 4) << '\n'; // 50
}
```

### Arithmetic Operations

- **`ptr + n`** yields a pointer to the element `n` positions after `ptr`.
- **`ptr - n`** yields a pointer `n` positions before `ptr`.
- **`ptr2 - ptr1`** returns the number of elements between two pointers (a `std::ptrdiff_t`). Both pointers must point into the same array; otherwise the behavior is undefined.
- Incrementing (`++ptr`) and decrementing (`--ptr`) move one element forward or backward.

### Subscript Equivalence

The subscript operator on arrays and pointers is just syntactic sugar for pointer arithmetic and dereference. The expression `arr[n]` is defined as `*(arr + n)`. Because addition is commutative, the bizarre expression `n[arr]` also compiles and means the same thing -- though you should never write that in real code.

This equivalence is why functions that receive a decayed pointer can still subscript it: the pointer behaves like an array from the caller's perspective.

### Iterating with Pointer Arithmetic

You can walk through an array with a pointer loop instead of an index loop. Use a pointer to the first element as the start, and a pointer one-past-the-last element as the sentinel:

```cpp
int* begin{ arr };
int* end{ arr + 5 };
for (int* p{ begin }; p != end; ++p)
    std::cout << *p << ' ';
```

This begin/end pattern is exactly what range-based `for` loops and the standard library algorithms use under the hood. Prefer `std::array` and range-based `for` in new code, but understanding pointer arithmetic is essential for reading legacy C and C++ codebases and for grasping how iterators actually work.
