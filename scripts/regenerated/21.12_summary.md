When you write `a = b` for objects of a class type, the compiler calls the **copy assignment operator** (`operator=`). If you do not provide one, the compiler generates a default that copies each member. For classes that need controlled copying, you can overload it yourself.

The copy assignment operator is a member function that takes a `const` reference to the same class type and returns a reference to `*this`. Returning `*this` by reference is essential -- it enables chained assignment like `a = b = c`, which evaluates right-to-left.

A correct implementation must handle **self-assignment** (the case where an object is assigned to itself, e.g. `x = x`). Without a guard, the function might reset a member before copying from it, corrupting the object. The standard idiom checks `this == &other` and returns early:

```cpp
#include <iostream>

class Fraction
{
    int m_num{};
    int m_den{ 1 };
public:
    Fraction(int n, int d) : m_num{n}, m_den{d} {}

    Fraction& operator=(const Fraction& other)
    {
        if (this == &other)
            return *this;
        m_num = other.m_num;
        m_den = other.m_den;
        return *this;
    }

    friend std::ostream& operator<<(std::ostream& out, const Fraction& f);
};
```

Do not confuse the copy assignment operator with the copy constructor. The copy constructor initializes a **new** object from an existing one (`Fraction f2{ f1 };` or `Fraction f2 = f1;`). The copy assignment operator is called when an **already-initialized** object appears on the left side of `=`. The distinction matters because construction and assignment are fundamentally different operations -- construction builds from scratch, while assignment overwrites existing state.

If the compiler-generated copy assignment (member-by-member copy) does the right thing for your class, prefer the default. Explicitly default it with `Fraction& operator=(const Fraction&) = default;` to signal intent. You only need a custom implementation when member-wise copy is insufficient -- a topic explored in the next lesson on shallow versus deep copy.
