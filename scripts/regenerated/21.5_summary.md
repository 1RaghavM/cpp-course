So far you have overloaded operators as friend functions or normal (non-member) functions. C++ also lets you overload operators as **member functions** of a class. When you do, the left-hand operand becomes the implicit `*this` object, so a binary operator that normally takes two parameters only needs one explicit parameter -- the right-hand operand.

```cpp
#include <iostream>

class Cents {
    int m_cents{};
public:
    Cents(int cents) : m_cents{ cents } {}
    Cents operator+(const Cents& rhs) const {
        return Cents{ m_cents + rhs.m_cents };
    }
    int getCents() const { return m_cents; }
};

int main() {
    std::cout << (Cents{6} + Cents{4}).getCents() << '\n';
}
```

When the compiler sees `a + b` and `a` is a `Cents` object, it rewrites the call as `a.operator+(b)`. Because `operator+` is a member, `a` is `*this` and `b` is the explicit parameter `rhs`. The member function can access `m_cents` on both objects directly since it is a member of the class.

The key limitation is that the left operand must always be an object of the class. If you need `int + Cents` (left operand is not your class), you cannot use a member function -- use a friend or normal function instead. This leads to a practical guideline for choosing the form:

- **Member function:** use when the left operand is always your class type and the operator modifies or inspects private state. Arithmetic operators like `+`, `-`, `*` between objects of the same type are common choices.
- **Friend function:** use when either operand could be a different type (e.g., `double * Cents`), when you need symmetric access, or when the left operand is not your class.
- **Normal (non-friend non-member) function:** use when the operator can be implemented entirely through the public interface, requiring no access to private members.

Not all operators can be overloaded both ways. The assignment (`=`), subscript (`[]`), function call (`()`), and member selection (`->`) operators **must** be overloaded as member functions. Conversely, the stream insertion (`<<`) and extraction (`>>`) operators **must** be non-member functions because their left operand is an `std::ostream` or `std::istream`, not your class.
