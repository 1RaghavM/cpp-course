## Timing Your Code

When comparing algorithms (e.g., selection sort vs. `std::sort`), you need a way to measure how long each one takes. The `<chrono>` library provides type-safe clocks and duration types for exactly this purpose.

```cpp
#include <chrono>
#include <iostream>

int main()
{
    auto start{ std::chrono::steady_clock::now() };

    // ... code you want to time ...

    auto end{ std::chrono::steady_clock::now() };
    auto elapsed{ end - start };
    auto ms{ std::chrono::duration_cast<std::chrono::milliseconds>(elapsed) };
    std::cout << "Took " << ms.count() << " ms\n";
}
```

### Clocks

`std::chrono::steady_clock` is the best choice for benchmarking because it is guaranteed to never go backwards (it is monotonic). `std::chrono::high_resolution_clock` may or may not be monotonic depending on the implementation, and `std::chrono::system_clock` can jump when the OS adjusts the system time. For measuring elapsed time, always prefer `steady_clock`.

### Time Points and Durations

Calling `steady_clock::now()` returns a `std::chrono::time_point`. Subtracting two time points produces a `std::chrono::duration`, which represents a span of time. A duration is parameterized by its tick period -- for example, `std::chrono::milliseconds` uses a period of 1/1000 of a second, and `std::chrono::microseconds` uses 1/1000000.

### duration_cast

Converting from a finer duration to a coarser one (e.g., microseconds to milliseconds) loses precision, so C++ requires an explicit `std::chrono::duration_cast`. Converting the other direction (coarser to finer) is implicit because no precision is lost. The `.count()` member function extracts the raw numeric value from any duration.

### Benchmarking Tips

Build in release mode (optimizations on) when benchmarking -- debug builds are dramatically slower. Run the timed section several times and look at the average. Avoid printing inside the timed section, since I/O is slow and variable. Be aware that the optimizer may remove code whose result is never used; make sure you use the computed result (e.g., print it after timing stops).
