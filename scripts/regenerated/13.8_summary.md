## Struct Aggregate Initialization

A struct with all public members and no user-declared constructors is an **aggregate**. C++ gives you three clean ways to initialize one, and choosing correctly avoids both bugs and verbosity.

### Brace initialization (list init)

Provide values inside braces in the same order the members are declared. Each value initializes the corresponding member positionally:

```cpp
#include <iostream>
#include <string>

struct Employee {
    std::string name;
    int id;
    double salary;
};

int main() {
    Employee e1 { "Alice", 101, 75000.0 };
    Employee e2 { "Bob", 102 };  // salary value-initialized to 0.0
    Employee e3 {};               // all members value-initialized

    std::cout << e1.name << ' ' << e1.salary << '\n';
    std::cout << e3.name.size() << ' ' << e3.id << '\n';
}
```

If you supply fewer values than there are members, the remaining members are **value-initialized** (zero for numeric types, empty for `std::string`). Supplying more values than members is a compile error.

### Designated initializers (C++20)

You can name the members you want to set using `.member =` syntax. This makes the initialization self-documenting and resilient to reading errors when a struct has many fields:

```cpp
Employee e4 { .name = "Carol", .id = 103, .salary = 88000.0 };
```

Designated initializers must appear in the same order the members are declared in the struct. You may skip members, and skipped ones are value-initialized. Mixing positional and designated initializers in the same braces is not allowed.

### Value initialization vs. default initialization

Using empty braces `Employee e{};` value-initializes every member: ints become 0, doubles become 0.0, strings become empty. Declaring without braces (`Employee e;`) **default-initializes**, which leaves fundamental types indeterminate. Always prefer braces to avoid reading garbage values.

### Copy-list initialization with `=`

You can also write `Employee e = { "Dana", 104, 62000.0 };`. The `=` is optional for aggregates. Both forms produce identical code. Prefer the shorter form without `=` for consistency.

### Assignment after creation

You can assign a new aggregate value to an existing struct variable using brace syntax: `e1 = { "Eve", 105, 91000.0 };`. This constructs a temporary and assigns it, replacing all members at once.
