## Introduction to Overloading the I/O Operators

In lesson 13.4 you wrote helper functions like `getColorName()` to convert enumerators to strings. That works, but it forces every print site to remember to call the helper: `std::cout << getColorName(shirt)`. Overloading `operator<<` lets you write `std::cout << shirt` directly — the compiler calls your function automatically.

### Overloading `operator<<`

`operator<<` is a binary operator. The left operand is `std::ostream&` (the stream) and the right operand is your type. Because the left operand is not your type, the overload must be a **free function** (not a member). It returns `std::ostream&` so calls can be chained (`std::cout << a << b`).

```cpp
#include <iostream>
#include <string_view>

enum Color { red, green, blue };

constexpr std::string_view colorToStr(Color c) {
    switch (c) {
        case red:   return "red";
        case green: return "green";
        case blue:  return "blue";
        default:    return "unknown";
    }
}

std::ostream& operator<<(std::ostream& out, Color c) {
    out << colorToStr(c);
    return out;
}

int main() {
    Color shirt{ green };
    std::cout << "Shirt color: " << shirt << '\n';
}
```

Key points: the return type is `std::ostream&`, the first parameter is `std::ostream& out` (not `std::cout` specifically — this works with any output stream), and you return `out` at the end.

### Overloading `operator>>`

The same idea applies to input. `operator>>` takes `std::istream&` on the left and your type (by non-const reference) on the right. Typically you read a value the stream already understands (like an `int` or `std::string`), then map it to your enumerator:

Read an integer, validate, and convert with `static_cast`. Return `in` so input calls can be chained. If the input is invalid, set the enumerator to a sensible default.

### Why This Matters

Once both operators are overloaded, your enum works seamlessly with `std::cout` and `std::cin` — no special helper calls at the use site. This is the idiomatic C++ approach and the pattern you will reuse for structs and classes later.

### Rules to Remember

- The overload must be a free function (the left operand is a standard library type).
- Always return the stream reference to support chaining.
- Keep the conversion logic in a separate helper (`constexpr` string-view function or switch) so the overload stays thin.
