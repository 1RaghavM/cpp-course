When you write a function template that uses operators like `+`, `<<`, or `==`, the compiler needs to find a matching operator overload for whatever type the template is instantiated with. If you have a class `Cents` with an overloaded `operator+`, calling a template function like `add(Cents{5}, Cents{10})` works -- the compiler instantiates the template with `Cents`, then resolves the `+` to your overload.

The tricky part is `operator<<`. If your overloaded `operator<<` is a non-member friend defined inside the class body, the compiler may fail to find it during template instantiation because of how name lookup works with templates. The fix is to ensure the operator is visible at namespace scope.

```cpp
#include <iostream>

template <typename T>
T add(T a, T b) { return a + b; }

class Cents {
    int m_cents{};
public:
    Cents(int c) : m_cents{c} {}
    Cents operator+(Cents rhs) const
    { return Cents{m_cents + rhs.m_cents}; }
    friend std::ostream& operator<<(
        std::ostream& out, const Cents& c)
    { out << c.m_cents; return out; }
};
```

With this setup, `add(Cents{5}, Cents{10})` returns a `Cents` and `std::cout << add(Cents{5}, Cents{10})` prints `15`. The `operator+` is found as a member function, and `operator<<` is found via argument-dependent lookup (ADL) because one of its arguments is of type `Cents`.

For templates with multiple type parameters, each operator must handle the exact types involved. A function template `template <typename T, typename U> T add(T a, U b)` would require that `operator+` accepts those two possibly-different types.

When a class itself is a template (e.g., `Wrapper<T>`), declaring a friend `operator<<` requires extra syntax. You must either define the friend function inline inside the class body (the simplest approach), or forward-declare both the class template and the operator template before the class definition. The inline approach avoids the forward-declaration complexity and is preferred for most cases.

The key rule: if your class will be used with function templates, make sure every operator the template might invoke is either a public member or discoverable via ADL. Test with a simple template call early -- operator lookup failures produce notoriously long error messages.
