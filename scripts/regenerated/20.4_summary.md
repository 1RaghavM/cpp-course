When you run a program from the terminal, you can pass extra information after the executable name — these are **command line arguments**. C++ gives you access to them through two special parameters on `main`:

```cpp
#include <iostream>
#include <string>
#include <sstream>

int main(int argc, char* argv[])
{
    std::cout << "You passed " << argc << " arguments:\n";
    for (int i{ 0 }; i < argc; ++i)
        std::cout << "  argv[" << i << "] = " << argv[i] << '\n';
    return 0;
}
```

Running `./program hello 42` prints three arguments: the program name (`argv[0]`), `"hello"` (`argv[1]`), and `"42"` (`argv[2]`).

**`argc`** (argument count) is the total number of arguments *including* the program name, so it is always at least 1. **`argv`** (argument vector) is an array of C-style strings (`char*`). You may also write it as `char** argv` — both forms are equivalent.

Because every argument arrives as a C-style string, you must convert explicitly when you need a number. The simplest modern approach is to wrap the argument in a `std::string` and use `std::stoi` (to `int`) or `std::stod` (to `double`). These functions throw `std::invalid_argument` if the string cannot be parsed, so production code should handle that case.

A few practical guidelines:

- Always check `argc` before accessing `argv[n]` to avoid reading out of bounds.
- `argv[argc]` is guaranteed to be a null pointer — you can use this as a sentinel in a loop, but checking `argc` is clearer.
- Arguments containing spaces must be quoted on the command line (`"hello world"` counts as one argument).
- The program name in `argv[0]` is implementation-defined; do not rely on its exact format.

Command line arguments are the standard way to make programs configurable without recompilation — file paths, flags, numeric parameters, and mode switches are all commonly passed this way.
