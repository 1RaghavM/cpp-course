# Keywords and Naming Identifiers

Keywords are reserved words in C++ that have special meaning to the compiler. You cannot use them as variable or function names. Common keywords you have already seen include `int`, `double`, `return`, and `const`. There are about 90 keywords in total — you will encounter more as you progress through this course.

Identifiers are names you create for variables, functions, and other entities. C++ is case-sensitive, so `myVar`, `MyVar`, and `myvar` are three different identifiers.

## Naming Rules

C++ enforces strict rules:
- Must start with a letter (a–z, A–Z) or underscore (`_`)
- Subsequent characters can be letters, digits (0–9), or underscores
- Cannot be a keyword or contain spaces
- Avoid starting with underscore followed by an uppercase letter or double underscores — these are reserved for compiler and standard library internals

## Naming Conventions

Following conventions makes code readable. The C++ Standard Library uses `snake_case` for variables and functions (`my_variable`), while many projects use `camelCase`. Pick one style and be consistent.

Choose descriptive names. `age` is better than `a`. Names should reflect what the variable represents. Abbreviations like `td` for `total_damage` hurt readability when you revisit code months later.

```cpp
#include <iostream>

int main()
{
    int player_health{ 100 };
    int player_damage{ 25 };
    int remaining_health{ player_health - player_damage };

    std::cout << "Remaining health: " << remaining_health << '\n';
    return 0;
}
```

Notice how `player_health`, `player_damage`, and `remaining_health` immediately tell you what each variable represents — no comments needed.

## The `std::` prefix revisited

From the iostream lesson: `std::cout` and `std::cin` live in the `std` namespace. The prefix `std::` is itself a naming convention — it prevents your identifiers from colliding with standard library names. Avoid `using namespace std;` because it pulls every standard library name into your scope, making collisions possible.

Good naming prevents compiler errors and makes your code maintainable. In C++, your variable names are often the primary documentation of intent.
