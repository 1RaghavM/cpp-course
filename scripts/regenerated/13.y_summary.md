## Using a Language Reference

At some point you will need a function you have never used before. Maybe you know the name, maybe you only know what you want to do. Either way, the answer lives in a **language reference** — a comprehensive catalog of every type, function, and header in the C++ standard library. The most widely used free reference is [cppreference.com](https://en.cppreference.com).

### Anatomy of a Reference Page

Every function page follows a predictable structure. Here is a condensed version of what you would see for `std::string::substr`:

```
constexpr basic_string substr( size_type pos = 0, size_type count = npos ) const;
```

Breaking this apart:

- **Return type** (`basic_string`) — tells you what you get back. For `std::string`, `basic_string` *is* `std::string`.
- **Parameters** — `pos` is where the substring starts (defaults to `0`); `count` is how many characters to extract (defaults to `npos`, meaning "until the end").
- **`const`** after the parameter list — the function does not modify the string you call it on.
- **`constexpr`** — usable in constant expressions (you can ignore this until it matters).

Below the signature you will find a description, a list of exceptions the function can throw, and — critically — any **undefined behavior** warnings (e.g., "the behavior is undefined if `pos > size()`"). Always read these.

### Translating a Signature into Code

Once you understand the signature, usage is mechanical:

```cpp
#include <iostream>
#include <string>

int main()
{
    std::string greeting{ "Hello, world!" };
    std::string sub{ greeting.substr(7, 5) }; // pos=7, count=5
    std::cout << sub << '\n';                 // prints: world
}
```

You matched `pos` to `7`, `count` to `5`, captured the returned `std::string`, and printed it. Every standard library call follows the same read-the-signature, fill-in-the-arguments pattern.

### Practical Tips

- **Search by name** when you know it (`std::string::find`). **Search by concept** when you do not ("C++ check if string contains substring").
- **Check the header** listed at the top of the page — that is the `#include` you need.
- **Read the "Parameters" and "Return value" sections first**, then skim "Exceptions" and "Notes" for gotchas.
- **Look at the example** at the bottom of every cppreference page — it is runnable code that demonstrates the function.
- **Pay attention to version tags** like `(since C++17)`. If a feature is tagged with a newer standard than you are compiling with, it will not compile.
