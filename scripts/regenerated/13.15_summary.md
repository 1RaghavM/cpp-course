## Alias Templates

In lesson 10.7 you learned that a **type alias** gives an existing type a new name using `using`. But a regular type alias cannot be templated -- if you want an alias for `std::pair<int, double>` and another for `std::pair<int, std::string>`, you must write each one by hand. **Alias templates** solve this by letting you create a family of aliases parameterized by one or more types.

An alias template is declared with `template` followed by a `using` statement:

```cpp
#include <iostream>
#include <utility>

template <typename T>
using IntPair = std::pair<int, T>;

int main()
{
    IntPair<double> a { 1, 3.14 };
    IntPair<std::string> b { 2, "hello" };
    std::cout << a.first << ' ' << a.second << '\n';
    std::cout << b.first << ' ' << b.second << '\n';
}
```

Here `IntPair<T>` is not a type itself -- it is a template that produces a type when you supply a template argument. `IntPair<double>` becomes `std::pair<int, double>`, and `IntPair<std::string>` becomes `std::pair<int, std::string>`. The compiler substitutes the argument into the right-hand side, exactly like a function template substitutes into its body.

### When Alias Templates Are Useful

Alias templates shine when a class template has multiple parameters but you want to fix some of them. Instead of repeating `std::pair<int, Something>` everywhere, `IntPair<Something>` communicates intent and reduces duplication. If you later change the fixed parameter from `int` to `long`, you update a single line.

They also help when a template name is long or deeply nested. A short alias template keeps call sites readable without sacrificing generality.

### Alias Templates vs. Regular Type Aliases

A regular type alias (`using Coord = std::pair<int, int>;`) fixes every parameter -- it names exactly one concrete type. An alias template leaves at least one parameter open, so it names a family of types. Use a regular alias when the type is fully known; use an alias template when the type still depends on a parameter the caller provides.

### Important Details

- Alias templates are not new types; they are just shorthand. `IntPair<double>` and `std::pair<int, double>` are the exact same type.
- CTAD (class template argument deduction) does not work through alias templates in C++17 or C++20. You must supply the template arguments explicitly.
- Like other templates, alias templates are typically defined in header files so every translation unit that uses them can see the definition.
