## Introduction to Iterators

An **iterator** is an object that points to an element in a container and lets you traverse the container without exposing how it stores its data. Every standard container (`std::array`, `std::vector`, etc.) provides iterators through the `begin()` and `end()` member functions. `begin()` returns an iterator to the first element; `end()` returns an iterator to one-past-the-last element (a sentinel that should never be dereferenced).

Three core operations make iterators work: **dereference** (`*it`) to read or write the pointed-to element, **increment** (`++it`) to advance to the next element, and **comparison** (`it != end`) to detect when traversal is complete.

```cpp
#include <array>
#include <iostream>

int main()
{
    std::array arr{ 10, 20, 30, 40 };

    for (auto it{ arr.begin() }; it != arr.end(); ++it)
        std::cout << *it << ' ';
    // prints: 10 20 30 40

    return 0;
}
```

You can also call the free functions `std::begin(c)` and `std::end(c)`, which work with any container and C-style arrays alike. For the standard containers we use in this course, the member-function form and the free-function form are interchangeable.

Because iterators abstract away indexing, the same loop pattern works for `std::vector`, `std::array`, and other containers without changing logic. Under the hood, a range-based `for` loop (`for (auto x : arr)`) is syntactic sugar that calls `begin()` / `end()` and uses exactly these iterator operations.

### Why not just use indices?

Indices require the container to support random access, and they couple your code to a specific container type. Iterators decouple traversal from storage, which becomes important when you use standard library algorithms later. They also eliminate signed/unsigned mismatch warnings that plague index-based loops.

### Key takeaways

- `begin()` points to the first element; `end()` points past the last.
- Dereference (`*it`), increment (`++it`), and compare (`it != end`) are the three fundamental iterator operations.
- Range-based `for` loops use iterators behind the scenes.
