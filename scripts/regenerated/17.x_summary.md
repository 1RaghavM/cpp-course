## Chapter 17 Summary

This chapter covered **fixed-size arrays** -- the modern `std::array` and the legacy C-style array -- plus the pointer arithmetic that underpins C-style array access.

### std::array

- `std::array<T, N>` wraps a fixed-length array. Initialize with braces; CTAD can deduce `T` and `N` from the initializers.
- `operator[]` is unchecked; `.at()` throws `std::out_of_range` on bad indices. `.size()` is `constexpr` and returns `std::size_t`.
- Pass by `const&` for read-only, by `&` to modify. Use a function template with `T` and `std::size_t N` to accept any size. Return by value -- the compiler applies copy elision.
- **Brace elision:** inner braces can be omitted when storing aggregates (structs) inside `std::array`.
- **`std::reference_wrapper`:** arrays cannot hold raw references; use `std::ref` / `std::cref` to store reference-like handles, accessed via `.get()` or implicit conversion.
- **Enum indices:** use a scoped enum to give indices meaningful names, casting to `std::size_t` for subscripting.

### C-style arrays

- Declared as `int arr[5];` with a compile-time constant size. Use `{}` to value-initialize.
- **Decay:** passed to a function, a C-style array converts to a pointer to its first element, losing size information. This is the main reason to prefer `std::array`.
- **Pointer arithmetic:** `ptr + n` advances by `n` elements. `arr[n]` is equivalent to `*(arr + n)`. Only valid within array bounds (including one-past-the-end).
- **C-style strings:** null-terminated `char` arrays. String literals decay to `const char*`. Prefer `std::string` / `std::string_view` in new code.
- **String constants:** `const char*` pointing to a literal acts as a symbolic constant. Identical literals may or may not share storage.
- **Multidimensional:** C-style: `int grid[3][4];` -- first dimension decays, rest must be specified. `std::array`: nest types like `std::array<std::array<int, 4>, 3>` for full type safety and `.size()` at every level.
