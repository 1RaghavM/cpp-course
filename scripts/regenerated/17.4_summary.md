## std::array of Class Types and Brace Elision

A `std::array` can hold any type that is copyable, including structs and classes. When the element type is an aggregate (like a struct with no user-declared constructors), each element itself needs braces, leading to a double-brace syntax:

```cpp
#include <array>
#include <iostream>

struct Point
{
    int x{};
    int y{};
};

int main()
{
    std::array<Point, 3> pts {{ {1, 2}, {3, 4}, {5, 6} }};
    for (const auto& p : pts)
        std::cout << p.x << ' ' << p.y << '\n';
    return 0;
}
```

The outer braces initialize the `std::array` object, the second set of braces initializes its internal C-style array member, and each `{1, 2}` initializes one `Point`. That is three levels of nesting.

### Brace Elision

C++ allows **brace elision**: the compiler can strip away inner brace levels during aggregate initialization when the meaning is unambiguous. This means a single pair of outer braces works too:

```cpp
std::array<Point, 3> pts { {1, 2}, {3, 4}, {5, 6} };
```

The compiler knows `Point` has two `int` members, so it consumes values two at a time. Both forms produce identical results. The single-brace version is more common in practice because it is easier to read.

### CTAD with std::array

Class template argument deduction (CTAD) lets the compiler deduce both the element type and size. However, CTAD requires all elements to have the same type. For class types, every element must be the same struct/class type and initialized the same way:

```cpp
std::array pts { Point{1, 2}, Point{3, 4} }; // deduces std::array<Point, 2>
```

With CTAD you must name the type inside each element's initializer because the compiler needs at least the first element's type to deduce the template parameter.

### When Double Braces Are Required

Brace elision does not always apply. If you value-initialize a `std::array` (all elements to zero/default), you still need double braces to avoid a compiler warning: `std::array<Point, 3> pts {{}};`. The inner `{}` value-initializes the internal array member. A single `{}` works functionally but some compilers warn about a missing brace level.
