## std::array and Enumerations

Back in lesson 16.9 you saw that enumerators can index arrays. Now that you know `std::array`, the combination becomes a powerful pattern for mapping enum values to associated data with compile-time safety.

### Using Enumerators as Indices

Because unscoped enumerators implicitly convert to integers, they work directly as `std::array` indices. This replaces error-prone magic numbers with self-documenting names:

```cpp
#include <array>
#include <iostream>
#include <string_view>

enum Animal { cat, dog, chicken, maxAnimals };

constexpr std::array<std::string_view, maxAnimals> animalNames{
    "Cat", "Dog", "Chicken"
};

int main() {
    for (int i{ 0 }; i < maxAnimals; ++i)
        std::cout << animalNames[static_cast<Animal>(i)] << '\n';
    return 0;
}
```

The `maxAnimals` enumerator equals the count of real enumerators (3), so it doubles as the array size. This is the **enum count idiom** -- add or remove an animal and both the enum and the array size update together, preventing mismatches.

### The Enum Count Idiom

Place a `max` or `count` enumerator at the end of your enum. Its implicit value equals the number of preceding enumerators (only when they start at 0 and increment by 1). Use it as the `std::array` size template argument. If someone adds a new enumerator before the count entry, the array size grows automatically -- but the compiler will not warn if you forget to add the corresponding data element. Keep the enum and the array initializer list close together to catch this visually.

### Mapping Enums to Data

This pattern creates a lookup table: define a `constexpr std::array` whose size is the count enumerator and whose elements correspond one-to-one with the real enumerators. Access it with `table[myEnumerator]`. Because the array is `constexpr`, lookups can be evaluated at compile time when the index is also `constexpr`.

For scoped enums (`enum class`), the implicit conversion does not exist. You must use `static_cast<std::size_t>(e)` or a helper function like `constexpr auto toIndex(MyEnum e) { return static_cast<std::underlying_type_t<MyEnum>>(e); }` to convert before indexing.

### When to Prefer This Over switch

A `switch` is better when each case performs different logic. An enum-indexed array is better when every enumerator maps to the same kind of data (a name, a price, a color code). The array version is shorter, eliminates repetitive `case` boilerplate, and the data is contiguous in memory.
