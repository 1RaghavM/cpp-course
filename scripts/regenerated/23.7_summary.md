When you write `int arr[]{1, 2, 3}`, the compiler knows how to initialize a built-in array from a braced list. But what if you want the same syntax for your own container class? C++ provides `std::initializer_list<T>` (from the `<initializer_list>` header) to make this possible.

An `std::initializer_list<T>` is a lightweight proxy object that gives access to an array of `const T` values created by a braced-init-list. It has `size()`, `begin()`, and `end()` member functions, so you can iterate over the values in a range-based `for` loop. The key detail is that the elements are **const** -- you cannot modify them through the initializer list.

To let your class accept brace initialization, add a constructor that takes `std::initializer_list<T>` by value (it is cheap to copy -- it only holds a pointer and a length, not the data itself).

```cpp
#include <initializer_list>
#include <iostream>

class IntArray {
    int m_size{};
    int* m_data{};
public:
    IntArray(std::initializer_list<int> list)
        : m_size{ static_cast<int>(list.size()) }
        , m_data{ new int[list.size()] } {
        int i{0};
        for (int v : list)
            m_data[i++] = v;
    }
    ~IntArray() { delete[] m_data; }
    int size() const { return m_size; }
    int operator[](int i) const { return m_data[i]; }
};
```

With this constructor, you can write `IntArray a{5, 10, 15, 20}` and the compiler constructs an `std::initializer_list<int>` from the braces and passes it to your constructor.

You can also provide an `operator=` that accepts `std::initializer_list<T>` to support list assignment after construction: `a = {100, 200}`. The implementation deletes the old storage, allocates new storage matching the list size, and copies the elements.

One important interaction with uniform initialization: if a class has both an `initializer_list` constructor and a regular constructor, **brace initialization strongly prefers the `initializer_list` constructor**. For example, if a class has `MyClass(int, int)` and `MyClass(std::initializer_list<int>)`, then `MyClass{5, 10}` calls the `initializer_list` version. To call the two-parameter constructor explicitly, use parentheses: `MyClass(5, 10)`. This preference is by design -- it makes brace-initialized lists behave consistently -- but it can be surprising if you are unaware of it.

`std::initializer_list` is what makes classes like `std::vector` accept `std::vector<int> v{1, 2, 3, 4}`. Understanding it completes the picture of how C++ containers support the same initialization syntax as built-in arrays.
