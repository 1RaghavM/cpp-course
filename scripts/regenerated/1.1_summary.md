# Statements and the Structure of a Program

A **statement** is the smallest independent unit of computation in C++. It tells the compiler to perform a specific action. Every statement in C++ ends with a semicolon (`;`), which acts as a terminator — similar to how a period ends a sentence.

## Types of statements

At this stage, the two most relevant kinds of statements are:

- **Declaration statements** — these create a variable and (optionally) give it an initial value. Example: `int x = 5;`
- **Expression statements** — these evaluate an expression and discard the result. Example: `x = 7;` (an assignment) or just `42;` (legal but pointless).
- **Return statements** — these pass a value back from a function. Example: `return 0;`

There are other statement types (compound statements, selection statements, iteration statements), but those will come later. For now, focus on the three above.

## The structure of a C++ program

Every C++ program must contain a function called `main`. When you run the compiled program (as you practiced in Chapter 0 — "Compiling your first program"), execution begins at the first statement inside `main` and proceeds downward, one statement at a time.

Here is a minimal valid C++ program:

```cpp
int main()
{
    int x = 0;
    int y = 5;

    x = y + 2;

    return 0;
}
```

Breaking this down:

- `int main()` declares the main function. `int` means it returns an integer.
- The curly braces `{` and `}` define a **compound statement** (also called a **block**) — they group multiple statements together into the body of the function.
- `int x = 0;` and `int y = 5;` are declaration statements. They create integer variables and set their values.
- `x = y + 2;` is an expression statement. It evaluates `y + 2` (which is `7`) and assigns the result to `x`.
- `return 0;` ends the function and passes `0` back to the operating system. By convention, `0` signals that the program ran successfully.

Statements execute in order, top to bottom. If you moved `x = y + 2;` above the line that creates `y`, the program would fail to compile because `y` would not exist yet.

Notice that every statement ends with a semicolon. Forgetting one is among the most common syntax errors you will encounter.
