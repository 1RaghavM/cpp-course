## Member Selection with Pointers and References

When you have a struct variable, you access its members with the **member selection operator** (`.`). But what happens when you have a *pointer* to a struct instead? You cannot write `ptr.member` because the pointer itself is not a struct -- it is an address. You first need to dereference it.

### Dereference Then Select vs. the Arrow Operator

You can dereference a pointer with `*` and then use `.`, but operator precedence requires parentheses because `.` binds tighter than `*`. Writing `(*ptr).member` everywhere is clunky, so C++ provides the **arrow operator** (`->`) as syntactic sugar. `ptr->member` is exactly equivalent to `(*ptr).member`. Use `->` whenever you access a member through a pointer -- it is the idiomatic choice and virtually all C++ code follows this convention.

```cpp
#include <iostream>
#include <string>

struct Employee {
    std::string name{};
    int id{};
};

void printViaPtr(const Employee* emp) {
    std::cout << emp->name << ' ' << emp->id << '\n';
}

void printViaRef(const Employee& emp) {
    std::cout << emp.name << ' ' << emp.id << '\n';
}

int main() {
    Employee e{ "Nara", 42 };
    Employee* ptr{ &e };
    ptr->name = "Sato";
    printViaPtr(ptr);
    printViaRef(e);
}
```

The arrow operator works with any pointer to a struct (or class) type, including pointers received as function parameters, as shown in `printViaPtr` above.

### Member Access Through References

References are not pointers -- a reference *is* the object it refers to. So you access members through a reference using the plain dot operator, just as you would with the original object. No arrow operator, no dereferencing. This is demonstrated by `printViaRef` in the example above.

### Quick Rule

If you have an object or a reference, use the dot operator (`.`). If you have a pointer, use the arrow operator (`->`). Prefer pass-by-const-reference over pass-by-address for functions that only read struct data, since references cannot be null and the call-site syntax is cleaner. Use pointers (and `->`) when you need nullable semantics or are working with dynamically allocated objects.
