This project ties together the operator overloading techniques from Chapter 21 by building a `Fraction` class from scratch. A fraction stores a numerator and denominator as integers, automatically reduces itself to lowest terms on construction, and supports a wide set of operators so it can be used as naturally as a built-in numeric type.

The project is split into two parts. In the first part you will build the core class with a constructor that reduces via greatest common divisor, arithmetic operators (`+`, `-`, `*`) implemented as friend functions, and I/O stream operators (`<<` and `>>`) so fractions can be printed and read in `numerator/denominator` format. The constructor must handle sign normalization -- a negative denominator should move the sign to the numerator so that `-1/3` and `1/-3` both store as `-1/3`.

In the second part you will extend the class with comparison operators (using `operator<=>` and `operator==` for full relational support), prefix and postfix `operator++` (which adds 1 to the fraction), and unary `operator-` to negate a fraction. The increment operators demonstrate the distinction between prefix (modify then return reference) and postfix (save copy, modify, return old copy), while the spaceship operator shows how a single function can replace six individual comparison operators.

The GCD algorithm is the linchpin of the design. Every constructor call and every arithmetic result must be reduced, ensuring that `2/4` is always stored as `1/2`. A simple iterative or recursive Euclidean algorithm works. You may use `std::gcd` from `<numeric>` if you prefer, but implementing it yourself is good practice.

Output format matters for the test cases. A fraction whose denominator is `1` after reduction should still print as `n/1` -- do not special-case whole numbers. A zero fraction prints as `0/1`. This keeps the output format uniform and the test cases deterministic.

By the end of this project your `Fraction` class will support expressions like `Fraction a{2, 3}; Fraction b{1, 6}; std::cout << (a + b) << '\n';` printing `5/6`, and comparisons like `if (a > b)` working correctly through the spaceship operator. This is a realistic example of how operator overloading makes user-defined types integrate seamlessly with the rest of the language.
