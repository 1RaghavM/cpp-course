## Chapter 13 Summary

This chapter introduced **program-defined types** -- custom types you create to model your problem domain.

### Enumerations

- An **unscoped enumeration** (`enum`) defines a type with a fixed set of named integral constants called enumerators. Enumerators are injected into the surrounding scope, which can cause naming collisions. They implicitly convert to their underlying integer type, but integers do not convert back without `static_cast`.
- A **scoped enumeration** (`enum class`) fixes both issues: enumerators are accessed via `EnumName::enumerator` and do not implicitly convert to integers. Prefer `enum class` unless you need implicit integer conversion.
- Convert enumerations to and from strings with helper functions, typically using a `switch` mapping each enumerator to its string representation.
- **Overloading `operator<<` and `operator>>`** for enum types enables direct I/O with `std::cout` and `std::cin`.

### Structs

- A **struct** groups related variables (members) under one type. Access members with the dot operator (`.`) or arrow operator (`->`) through a pointer.
- **Aggregate initialization** uses braces in declaration order: `Point p{ 3.0, 4.0 };`. C++20 adds **designated initializers**: `Point p{ .x = 3.0, .y = 4.0 };`.
- **Default member initializers** provide fallback values in the struct definition. If an initializer list omits a member, its default applies.
- Structs can be **passed to and returned from functions**. Pass by `const` reference to avoid copies; return by value for small structs.
- Structs support nesting and **structured bindings** (`auto [a, b] = myStruct;`) for decomposing members.
- **Member selection with pointers**: use `ptr->member` instead of `(*ptr).member`.

### Class Templates

- A **class template** parameterizes a struct definition by one or more types via `template<typename T>`. The compiler generates a concrete type for each unique set of template arguments.
- **CTAD** (class template argument deduction) lets the compiler deduce template arguments from initializer values, so `Pair p{ 1, 2.5 };` works without explicit `Pair<int, double>`.
- An **alias template** creates a shorthand for a template instantiation: `template<typename T> using Name = SomeTemplate<T, T>;`.
