# Introduction to Objects and Variables

In the previous lessons you learned that a C++ program is a sequence of statements inside `main`, and that comments let you annotate code without affecting its behavior. So far, the only data your programs have worked with is the literal number in `return 0;`. Real programs need to store, retrieve, and manipulate data. That is what objects and variables are for.

## RAM and data storage

When your program runs, the operating system gives it a region of **RAM** (random-access memory) to work with. RAM is fast, temporary storage — data in RAM disappears when the program ends. Each piece of data your program needs occupies some number of bytes in this memory. You do not need to manage individual bytes yourself; C++ handles the details when you create variables.

## Objects and variables

An **object** is a region of memory that can hold a value. A **variable** is a named object — you give it a name so you can refer to it later in your code.

To create a variable, you write a **definition** (also called a **declaration** for simple cases). The definition tells the compiler two things: the **type** of data the variable holds and the **name** you want to use.

## The `int` type

You have already seen `int` in `int main()`. The keyword `int` is a **type** that represents a whole number (an integer). When you define an `int` variable, the compiler reserves enough memory (typically 4 bytes) to store an integer value.

## Defining and initializing variables

Here is a small program that defines several `int` variables and uses one in the return statement:

```cpp
int main()
{
    int width = 10;    // definition with initialization
    int height = 3;    // another initialized variable
    int area = width * height; // initialized using an expression

    return area; // exit code will be 30
}
```

Each line like `int width = 10;` does three things: declares a variable named `width`, allocates memory for an `int`, and stores the value `10` into that memory. The variable `area` is initialized from an expression that multiplies the values of `width` and `height`.

## Variable names

A variable name (also called an **identifier**) must start with a letter or underscore and can contain letters, digits, and underscores. Names are case-sensitive — `count` and `Count` are different variables. Lesson 1.7 covers naming conventions in more detail; for now, stick to short, descriptive lowercase names.
