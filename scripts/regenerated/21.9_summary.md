The **subscript operator (`operator[]`)** lets objects of your class use array-style indexing syntax. It must be overloaded as a member function — it cannot be a friend or free function.

The parameter inside the brackets is whatever type makes sense for your container. For a simple integer-indexed list, the parameter is an `int` or `std::size_t`. The return type is almost always a **reference** so the caller can both read and assign through the subscript:

```cpp
#include <iostream>

class IntArray {
    int m_data[5]{};
public:
    int& operator[](int index) { return m_data[index]; }
    int operator[](int index) const { return m_data[index]; }
};

int main() {
    IntArray arr;
    arr[2] = 42;
    std::cout << arr[2] << '\n'; // 42
    return 0;
}
```

The non-const overload returns `int&`, which allows expressions like `arr[2] = 42` to compile — the reference binds to the actual element, so the assignment modifies the stored value. The const overload returns by value (or `const int&`) and is selected when the object itself is const. Providing both overloads is the standard practice: non-const objects get read-write access, and const objects get read-only access.

Because `operator[]` forwards directly to an internal array or data structure, it is the natural place to add **bounds checking**. You can assert or throw if the index is out of range, catching bugs that raw array access would silently allow.

A few rules to remember. `operator[]` takes exactly one parameter (the index inside the brackets). It must be a member function — the compiler requires this because the left-hand operand is always the object being indexed. The return-by-reference pattern is what makes subscript assignment work; without it, `arr[i] = value` would assign to a temporary and the stored element would remain unchanged.
