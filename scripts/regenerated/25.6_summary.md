When you declare a virtual function, the compiler must decide which version to call at runtime. It does this through a hidden data structure called the **virtual table** (vtable).

Every class that declares or inherits a virtual function gets its own vtable -- a static array of function pointers, one entry per virtual function. `Base` has a vtable where entry 0 points to `Base::speak`. `Derived` has its own vtable where entry 0 points to `Derived::speak` instead. If `Derived` does not override a particular function, its vtable entry still points to the base version.

Every object of such a class contains a hidden **vptr** (virtual table pointer). The compiler inserts this automatically. When you create a `Derived` object, the constructor sets the vptr to point at `Derived`'s vtable. When you call a virtual function through a `Base*`, the runtime follows the vptr to the vtable, looks up the function pointer at the right slot, and calls it. This is the mechanism behind late binding.

The vptr has a measurable cost: it adds one pointer (8 bytes on a 64-bit system) to every object. A class with a single `int` and no virtual functions occupies 4 bytes; add one virtual function and it grows to 16 bytes (8 for the vptr, 4 for the int, 4 bytes padding).

```cpp
#include <iostream>
struct Plain  { int x{}; };
struct Virtual { int x{}; virtual void f() {} };
int main() {
    std::cout << sizeof(Plain) << '\n';    // 4
    std::cout << sizeof(Virtual) << '\n';  // 16
    return 0;
}
```

Virtual calls also have a slight runtime cost: one extra pointer dereference plus an indirect function call. In practice this is tiny, but it means you should only mark functions virtual when you need runtime dispatch.

Key points:

- One vtable per class (shared by all objects), one vptr per object
- Adding more virtual functions grows the vtable but not the object -- the vptr size is fixed
- The vptr is set during construction, so calling a virtual function from a constructor dispatches to the current class's version, not a derived override
- The vtable is a compiler implementation detail, not part of the C++ standard, but virtually all compilers use this approach
