## Introduction to Structs, Members, and Member Selection

In lesson 13.1 you saw that program-defined types let you model real concepts. A **struct** is the most common way to bundle related variables into a single type. Each variable inside a struct is called a **member** (or **data member**, or **member variable**).

### Defining a Struct

A struct definition begins with the `struct` keyword, followed by the type name (PascalCase by convention), then the member declarations inside braces. A semicolon after the closing brace is required — forgetting it is a classic compiler error.

This definition creates a new type. No object exists yet — you have only described what an object of that type looks like. Each object you later create will contain its own independent copy of every member.

### The Member Selection Operator (dot operator)

To access a specific member of a struct object, use the **member selection operator** (`.`), also called the **dot operator**. The left operand is the struct object and the right operand is the member name:

```cpp
#include <iostream>

struct Employee {
    int id;
    double salary;
}; // semicolon required after closing brace

int main() {
    Employee alice; // creates an Employee object
    alice.id = 1;
    alice.salary = 75000.0;
    std::cout << alice.id << " earns "
              << alice.salary << '\n';
}
```

The dot operator is one of the highest-precedence operators in C++, so expressions like `alice.salary * 0.1` work without extra parentheses.

### Members Are Ordinary Variables

Once selected via the dot operator, a member behaves exactly like any standalone variable of the same type. You can assign to it, pass it to functions, use it in expressions, and read from it. The struct simply provides organizational grouping — it does not change how the underlying types work.

### Each Object Gets Its Own Members

When you create two `Employee` objects, each has its own `id` and `salary` in memory. Modifying one object's members has no effect on the other. This is no different from how two `int` variables are independent — structs just let you group several related values under one name.

### Why Structs Over Separate Variables

Without structs, representing an employee means juggling separate `id` and `salary` variables that have no formal relationship. Passing them to a function requires two parameters instead of one. Structs solve this: one object, one parameter, one return value — and the compiler enforces that `id` and `salary` always travel together.
