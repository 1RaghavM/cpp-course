A **pointer to a pointer** (declared with `**`) holds the address of another pointer. When you dereference once with `*ptr`, you get the inner pointer; dereference again with `**ptr` to reach the underlying value. This concept is essential for dynamically allocating two-dimensional arrays.

To create a dynamic 2D array, you allocate an array of `int*` pointers (one per row), then allocate each row individually. This gives you a contiguous array of row pointers, where each row is its own heap-allocated array.

```cpp
#include <iostream>

int main()
{
    int rows{ 3 };
    int cols{ 4 };
    int** matrix{ new int*[rows] };
    for (int r{ 0 }; r < rows; ++r)
        matrix[r] = new int[cols]{};

    matrix[1][2] = 42;
    std::cout << matrix[1][2] << '\n';

    for (int r{ 0 }; r < rows; ++r)
        delete[] matrix[r];
    delete[] matrix;
}
```

The cleanup order matters: you must `delete[]` each row first, then `delete[]` the array of row pointers. Reversing this order loses access to the row pointers and leaks every row. Think of it as unwinding in the opposite order of allocation.

Because each row is allocated separately, the rows are not necessarily contiguous in memory. This is different from a fixed-size 2D C-style array where all elements occupy a single block. The tradeoff is flexibility -- each row could even have a different length (a "jagged" array) -- but you lose cache locality.

Declaring a pointer to a pointer follows normal pointer syntax stacked one level deeper: `int** ptr` means "ptr is a pointer to a pointer to int." You can chain more levels (`int***`), but in practice two levels is the most you will encounter. Beyond that, the code becomes hard to reason about and better alternatives exist (like `std::vector<std::vector<int>>`).

Key rules to remember: always pair every `new` with a `delete` and every `new[]` with a `delete[]`. For a dynamic 2D array with `r` rows, that means `r + 1` total allocations and `r + 1` total deallocations.
