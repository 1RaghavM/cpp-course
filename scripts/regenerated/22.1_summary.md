Chapter 19 introduced dynamic memory allocation with `new` and `delete`. While powerful, manual memory management is one of the most common sources of bugs in C++. This lesson examines the core problems and introduces the concept of **ownership** -- the foundation that the rest of this chapter builds on.

Three recurring issues plague raw-pointer-based resource management:

- **Memory leaks:** Forgetting to call `delete`, or having an early `return` or exception skip the `delete` statement entirely.
- **Dangling pointers:** Using a pointer after the memory it points to has been freed.
- **Unclear ownership:** When multiple pointers refer to the same heap object, it becomes ambiguous which part of the code is responsible for deleting it.

Consider a function that allocates, uses, and frees memory:

```cpp
#include <iostream>

void process(int value) {
    int* ptr{ new int{ value * 2 } };
    if (value == 0)
        return; // leak: ptr is never deleted
    std::cout << *ptr << '\n';
    delete ptr;
}

int main() {
    process(5);  // ok
    process(0);  // leaks memory
}
```

The early `return` skips `delete`, leaking memory. The Rule of Three (Chapter 21) helps classes manage their own resources, but it does not solve the problem of bare pointers in functions or of transferring ownership between objects.

**Ownership** means exactly one entity is responsible for the lifetime of a resource. If you can always identify who owns a heap allocation, you know exactly where `delete` should happen. The difficulty is that raw pointers carry no ownership information -- a `T*` might own the memory, or it might just be observing it.

C++ solves this with two mechanisms introduced in the rest of this chapter:

- **Smart pointers** -- class templates that wrap a raw pointer and automatically call `delete` when the owning object is destroyed. They encode ownership directly in the type system.
- **Move semantics** -- a language feature that allows transferring ownership of resources from one object to another without copying. This makes smart pointers efficient to return from functions and store in containers.

Both build on concepts you already know: destructors, operator overloading, and dynamic allocation. The following lessons cover each in detail.
