In lesson 20.1, you learned about function pointers and how they let you pass behavior as an argument. The downside is that you must define a named function somewhere, even if it is only used once. **Lambdas** (also called anonymous functions) solve this: they let you define a function right where you need it.

The basic lambda syntax is:

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main()
{
    std::vector v{ 5, 2, 8, 1, 9, 3 };

    std::sort(v.begin(), v.end(),
        [](int a, int b) { return a > b; }); // sort descending

    for (auto val : v)
        std::cout << val << ' ';             // 9 8 5 3 2 1
}
```

The three parts of a lambda are: `[]` (the capture clause -- empty for now, meaning the lambda uses no outside variables), `(int a, int b)` (the parameter list, just like a regular function), and `{ return a > b; }` (the body).

When the body is a single `return` statement, the compiler deduces the return type automatically. You can also specify it explicitly with a **trailing return type**: `[](int x) -> double { return x * 1.5; }`. Use the explicit form when the deduced type is not what you want or when the body has multiple return paths with different types.

You can store a lambda in a variable using `auto`, which lets you reuse it or give it a readable name:

```cpp
auto isEven{ [](int n) { return n % 2 == 0; } };
std::cout << isEven(4);  // 1 (true)
```

Lambdas work anywhere a callable is expected. You have already used `std::sort` and `std::for_each` with function pointers -- lambdas drop in as a replacement with no extra syntax. They also work with `std::count_if`, `std::find_if`, `std::all_of`, and the other `<algorithm>` functions that accept a predicate or comparison.

Because the capture clause is empty `[]`, these lambdas can only use their parameters and local variables defined inside the body. Accessing variables from the enclosing scope requires captures, which is the topic of the next lesson.
