Unary operators act on a single operand. When you overload `operator+`, `operator-`, or `operator!` as **member functions**, they take no parameters because the object itself (`*this`) is the operand. The compiler translates an expression like `-obj` into `obj.operator-()`.

All three follow the same pattern: return a new value computed from the current object's state without modifying the original. `operator-` typically returns a new object with negated members, `operator+` often returns a copy unchanged (useful for symmetry or explicit sign), and `operator!` returns a `bool` representing the logical negation of some condition the class defines.

```cpp
#include <iostream>

class Point {
    int m_x{}, m_y{};
public:
    Point(int x, int y) : m_x{x}, m_y{y} {}
    Point operator-() const { return Point{-m_x, -m_y}; }
    Point operator+() const { return *this; }
    bool operator!() const { return (m_x == 0 && m_y == 0); }
    int getX() const { return m_x; }
    int getY() const { return m_y; }
};
```

Key details to remember. First, mark these functions `const` -- they should not modify the object they operate on. Second, `operator-` and `operator+` return the class type by value (a new object), while `operator!` returns `bool`. Third, do not confuse unary minus (`-obj`, no parameters) with binary minus (`a - b`, one parameter); the compiler distinguishes them by parameter count.

Unary `operator+` rarely changes behavior, but defining it keeps your interface symmetric when you provide `operator-`. Unary `operator!` is most useful when your class has a natural notion of "empty" or "zero" -- it lets objects appear naturally in boolean contexts like `if (!obj)`.

Because these operators do not modify the object, they never return a reference to `*this`. They always produce a fresh value, which is why the return is by value for the class type and by value for `bool`.
