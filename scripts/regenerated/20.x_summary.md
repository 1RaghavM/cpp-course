## Chapter 20 Summary

This chapter covered advanced function topics: treating functions as data, memory layout during calls, self-calling functions, and inline function objects.

### Function pointers

- A function pointer stores a function's address and calls it indirectly. Syntax: `int (*funcPtr)(int, int)` -- parentheses around `*funcPtr` are mandatory.
- Enable callback patterns: pass a function as an argument so the callee can invoke caller-chosen behavior.
- A function implicitly converts to a pointer to itself. Default arguments do not work through function pointers.

### The stack and the heap

- The **call stack** stores local variables and call frames. Each call pushes a frame; each return pops it. Fast but limited in size.
- The **heap** is where dynamic allocations live. Persists until explicitly freed, much larger than the stack, but slower to allocate.
- Stack overflow occurs from deep recursion or oversized local arrays.

### Recursion

- A recursive function calls itself and needs a **base case** to terminate. Each call gets its own stack frame.
- Naturally expresses problems with self-similar substructure: factorial, Fibonacci, tree traversal.
- Prefer iteration when performance matters; C++ does not guarantee tail-call optimization.

### Command line arguments

- `int main(int argc, char* argv[])` receives arguments as C-style strings. `argv[0]` is the program name; user arguments start at `argv[1]`.
- Numeric arguments must be converted with `std::stoi` or `std::stod`.

### Ellipsis

- C-style ellipsis (`...`) accepts variable arguments but bypasses all type checking, making it unsafe. Prefer variadic templates or `std::initializer_list`.

### Lambdas

- Syntax: `[captures](parameters) { body }`. Return type is usually deduced.
- Store with `auto` or pass directly to algorithms like `std::sort` and `std::count_if`.
- Capture clause: `[x]` by value, `[&x]` by reference, `[=]` all by value, `[&]` all by reference, or mix them.
- Value captures are const by default; add `mutable` to modify the copy.
- Captureless lambdas implicitly convert to function pointers.
