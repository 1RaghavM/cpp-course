## Chapter 21 Summary

This chapter covered operator overloading: defining custom behavior for C++ operators when applied to user-defined types.

- **Introduction (21.1):** Operators are functions. You can overload them when at least one operand is a user-defined type. You cannot change arity, precedence, or associativity. Overloads can be friend functions, normal non-member functions, or member functions.

- **Arithmetic via friend functions (21.2):** A `friend` function declared inside the class accesses private members. Implement compound assignment (`+=`) first, then define the binary operator in terms of it.

- **Arithmetic via normal functions (21.3):** When public getters suffice, prefer non-friend non-member functions to reduce coupling.

- **I/O operators (21.4):** `operator<<` and `operator>>` must be non-member functions (left operand is a stream). They return a stream reference for chaining and are often friends for private access.

- **Member function overloads (21.5):** The implicit `*this` is the left operand, so binary member operators take one explicit parameter. `=`, `[]`, `()`, and `->` must be member functions.

- **Unary +, -, ! (21.6):** `operator-` returns a negated copy, `operator+` typically returns unchanged, and `operator!` returns a bool for logical negation.

- **Comparison operators and <=> (21.7):** Defaulting `operator<=>` in C++20 auto-generates all six comparison operators. It returns `std::strong_ordering`, `std::weak_ordering`, or `std::partial_ordering`.

- **Increment/decrement (21.8):** Prefix (`operator++()`) returns a reference to the modified object. Postfix (`operator++(int)`) takes a dummy `int`, saves state, increments, and returns the saved copy.

- **Subscript operator (21.9):** `operator[]` must be a member function, typically returning a reference. Provide a `const` overload for const objects.

- **Parenthesis operator / functors (21.10):** `operator()` can take any parameters. Objects with it are **functors**--they hold state between calls and work as predicates in standard algorithms.

- **Typecasts (21.11):** Written as `operator TargetType() const` with no explicit return type. Mark `explicit` to prevent implicit conversions; `static_cast` and boolean contexts still work.

- **Assignment operator (21.12):** Must be a member, returns `*this` by reference. Handle self-assignment with `if (this == &other)`. Write a custom one when managing dynamic resources.

- **Shallow vs. deep copying (21.13):** Shallow copies share pointed-to memory (dangling pointer risk). Deep copies allocate independently. Classes managing dynamic memory need custom copy constructors and assignment operators.

- **Operators with templates (21.14):** Operator overloads work with class templates. A friend defined inside a class template is a per-instantiation non-template function. Friend function templates require forward declarations.
