## Function Parameters and Arguments

In "Introduction to functions" you learned how to define and call functions. So far those functions always did the same thing every time. **Parameters** let you pass data *into* a function so it can work on different values each call.

### Parameters vs. arguments

- A **parameter** is a variable declared in the function signature. It lives only for the duration of that function call.
- An **argument** is the concrete value you supply at the call site -- it gets copied into the parameter.

```cpp
#include <iostream>

// 'a' and 'b' are parameters
int add(int a, int b) {
    return a + b;
}

int main() {
    std::cout << add(2, 8) << '\n';  // arguments: 2 and 8 -> prints 10
    std::cout << add(3, 4) << '\n';  // arguments: 3 and 4 -> prints 7
    return 0;
}
```

### Pass by value

C++ copies arguments into parameters -- this is called **pass by value**. Modifying a parameter inside the function has no effect on the variable the caller used. If you come from Python, this matches how Python handles integers and other immutable types.

### Multiple parameters

Separate parameters with commas. Each parameter needs its own type declaration -- you cannot write `int a, b` as a shorthand. Write `int a, int b`.

```cpp
int multiply(int x, int y) {
    return x * y;
}
```

### Parameters vs. return values

Remember from "Function return values (value-returning functions)": a `return` statement sends data *out* of a function. Parameters send data *in*. Together they let functions act as self-contained, reusable transformations: accept inputs, produce output, change nothing else.

### Void functions with parameters

"Void functions (non-value returning functions)" showed that not every function needs to return something. A void function can still accept parameters -- for example, a `void printSum(int a, int b)` function produces side-effects (printing) without returning a value.

### Key rules

- Declare every parameter with an explicit type.
- Parameter names are local to the function body; name collisions with `main` variables are harmless.
- The order of arguments must match the order of parameters.
- Pass by value means the caller's variables are never changed by the function unless you later learn about references or pointers.
