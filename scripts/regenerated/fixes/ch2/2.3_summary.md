## Void Functions (Non-Value Returning Functions)

In "Function return values (value-returning functions)" you saw how functions can compute and return a result the caller uses directly. But many functions exist purely to *do something* -- print output, modify state, or coordinate work -- and have no meaningful value to hand back. For these, C++ provides the `void` return type.

### Declaring a Void Function

Replace the return type with `void`. The function body may contain an explicit `return;` (no value) to exit early, or simply fall off the end -- both are valid.

```cpp
#include <iostream>

void printSum(int a, int b) {
    int total { a + b };
    std::cout << "Sum: " << total << '\n';
}

int main() {
    printSum(3, 4);   // Sum: 7
    printSum(10, 20);  // Sum: 30
    return 0;
}
```

`printSum` performs a **side effect** -- writing to the console -- instead of producing a value. The caller cannot write `int s = printSum(3, 4);` because `void` carries no data.

### Side Effects vs. Return Values

A **side effect** is any observable change outside a function's own local scope: printing, writing to a file, modifying a variable passed by reference, or updating global state. Void functions are the natural home for side effects.

Choose `void` when:
- The function's entire purpose is its side effect (printing, logging, drawing).
- There is no single meaningful result to return.

Choose a return type when:
- The caller needs the computed value to make a decision or do further work (see "Function return values").
- You want the function to stay pure -- same inputs always produce the same output with no side effects.

### Python Comparison

In Python every function implicitly returns `None` when no `return` statement is present. C++ makes the intent explicit: `void` tells both the compiler and the reader that no return value exists. Trying to `return 42;` from a `void` function is a compile error in C++.

### Early Return from Void

Use a bare `return;` to exit early. This is sometimes used for guard clauses -- you will see examples of this pattern once you learn conditional statements in a later chapter. For now, know that a void function can contain `return;` to stop running before reaching the end of its body.

### Combining void and value-returning functions

A common pattern is to have value-returning functions compute results, and void functions handle all the printing:

```cpp
#include <iostream>

int multiply(int a, int b) {
    return a * b;
}

void printResult(int value) {
    std::cout << "Result: " << value << '\n';
}

int main() {
    int product { multiply(6, 7) };
    printResult(product);  // Result: 42
    return 0;
}
```

Keep functions focused: a void function should perform one clear action, just as a value-returning function should compute one clear result.
