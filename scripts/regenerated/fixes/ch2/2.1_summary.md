## Introduction to Functions

A **function** is a named block of code that performs a specific task. You have already been using one function: `main`. Every C++ program must define `main` -- it is where execution begins. Now you will write your own.

### Your first function

```cpp
#include <iostream>

// Function definition
void printWelcome() {
    std::cout << "Welcome to C++!" << '\n';
}

int main() {
    printWelcome();  // function call
    printWelcome();  // call it again — same output, no duplicated code
    return 0;
}
```

Breaking down the definition:

- **Return type** (`void`) -- the type of value the function sends back to the caller. `void` means "returns nothing." You will learn about functions that return values in lesson 2.2.
- **Function name** (`printWelcome`) -- follows the same naming rules as variables: lowercase, descriptive.
- **Parameter list** (`()`) -- currently empty. You will learn how to pass data into functions in lesson 2.4.
- **Body** -- the block `{ ... }` containing the statements that run when the function is called.

### Calling a function

Writing `printWelcome()` is a **function call**. When C++ reaches a call, it jumps to that function's body, executes all of its statements, and then returns to the line after the call.

```cpp
#include <iostream>

void printHeader() {
    std::cout << "=== Report ===" << '\n';
}

void printFooter() {
    std::cout << "=== End ===" << '\n';
}

int main() {
    printHeader();
    std::cout << "Data goes here" << '\n';
    printFooter();
    return 0;
}
```

### Why functions?

1. **Avoid repetition** -- write logic once, call it many times.
2. **Readability** -- a well-named function makes the caller's intent obvious without reading the implementation.
3. **Testability** -- small, focused functions are easier to verify in isolation.

### Key differences from Python

Python infers types at runtime; C++ requires you to state the return type at compile time. There is no `def` keyword -- the return type itself signals the start of a function definition. C++ also requires a function to be **declared before it is used** (covered in lesson 2.7 on forward declarations).

Start every function with a single, clear responsibility. If you find yourself writing a function that does two unrelated things, split it.
