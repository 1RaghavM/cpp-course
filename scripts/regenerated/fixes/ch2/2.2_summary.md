## Function Return Values

In "Introduction to functions" you saw how to define and call functions. Those functions did their work and returned to the caller without sending any data back. **Value-returning functions** change that -- they compute something and hand it back to whoever called them.

### The return type

Every function declaration begins with a **return type**, placed before the function name:

```cpp
#include <iostream>

// Returns the sum of two integers
int add(int a, int b) {
    return a + b;
}

int main() {
    int result { add(7, 3) };
    std::cout << result << '\n'; // 10
    return 0;
}
```

`int` before `add` tells the compiler -- and the reader -- that this function produces an `int`. The `return` statement exits the function and passes its expression back to the caller.

### `return` ends execution immediately

As soon as a `return` statement runs, the function stops. Any code after it in the same path is unreachable.

```cpp
int triple(int x) {
    return x * 3;
    // anything here would never execute
}
```

### `main` is special

`main` must be declared `int` and return a status code to the operating system. `return 0` signals success. You have seen this in every program so far. The compiler inserts an implicit `return 0` at the end of `main` if you omit it, but writing it explicitly is good practice.

### `void` -- when there is nothing to return

If a function does not produce a value, declare its return type as `void` and omit the `return` statement (or use a bare `return;` to exit early). You cannot use the result of a `void` function in an expression. Void functions are covered in detail in the next lesson.

### Using return values

A return value can be stored in a variable, passed directly to another function, or used in an expression:

```cpp
int square(int x) { return x * x; }

int main() {
    // nested call: square the result of add
    std::cout << square(add(2, 3)) << '\n'; // 25
    return 0;
}
```

In Python you may have written functions that return `None` by default; in C++ every non-`void` function **must** reach a `return` statement that provides a value of the declared type, or the behavior is undefined. The compiler will warn -- often error -- if a code path is missing a return.

### Key takeaways

- The return type comes first in the function signature.
- `return <expr>;` sends the value back and exits the function immediately.
- `main` returns `int`; `return 0` means success.
- `void` functions produce no value and need no `return` expression.
