You already know that pointers store the address of a variable. It turns out that functions also have addresses -- when a program is compiled, each function lives at a specific location in memory. A **function pointer** is a variable that holds the address of a function, letting you call that function indirectly.

To declare a function pointer, the syntax mirrors the function's signature with an extra `(*)` where the name goes. For a function `int add(int, int)`, the matching pointer type is `int(*)(int, int)`. The parentheses around `*` are mandatory -- without them the compiler reads it as a function returning `int*`.

```cpp
#include <iostream>

int add(int a, int b) { return a + b; }
int multiply(int a, int b) { return a * b; }

int applyOp(int x, int y, int(*op)(int, int))
{
    return op(x, y);
}

int main()
{
    int(*fn)(int, int){ &add };
    std::cout << fn(3, 4) << '\n';            // 7
    std::cout << applyOp(3, 4, multiply) << '\n'; // 12
    return 0;
}
```

A function's name implicitly converts to a pointer to that function, so `&add` and `add` both work when assigning to a function pointer. Calling through a function pointer works the same as a normal call -- just use the pointer name with parentheses and arguments. An explicit dereference like `(*fn)(3, 4)` also works but is unnecessary.

The real power of function pointers is the **callback pattern**: passing a function pointer as a parameter so the called function can invoke behavior chosen by the caller. The `applyOp` function above does not know which operation to perform -- the caller decides by passing `add` or `multiply`.

Because the declaration syntax is verbose, a **type alias** makes code easier to read:

```
using ArithmeticFn = int(*)(int, int);
```

Now you can write `ArithmeticFn fn{ &add };` or use `ArithmeticFn` as a parameter type. Prefer type aliases whenever a function pointer type appears more than once.

Function pointers must match the target function's return type and parameter types exactly. Assigning an incompatible function is a compile error. A function pointer can also be set to `nullptr`, meaning it points to no function -- dereferencing a null function pointer is undefined behavior.
