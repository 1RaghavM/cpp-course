## Introduction to std::array

You already know `std::vector`, which can grow and shrink at runtime. When the number of elements is known at compile time and will never change, `std::array` (from `<array>`) is the better choice. It has a fixed size baked into its type, uses stack memory instead of heap allocation, and carries zero overhead compared to a raw C-style array.

### Declaring and Initializing

`std::array` is a class template that takes two template arguments: the element type and the number of elements. You initialize it with a brace-enclosed list, just like `std::vector`:

```cpp
#include <array>
#include <iostream>

int main()
{
    std::array<int, 4> scores{ 90, 85, 72, 98 };

    scores[2] = 75;                  // modify third element
    std::cout << scores[0] << '\n';  // 90
    std::cout << scores[2] << '\n';  // 75
}
```

If you provide fewer initializers than the array size, the remaining elements are value-initialized (zero for numeric types). An empty initializer list `{}` zeros the entire array.

### Accessing Elements

`operator[]` accesses an element by index, starting from `0`, exactly like `std::vector`. No bounds checking is performed, so an out-of-range index is undefined behavior. The `.at()` member function does check bounds and throws `std::out_of_range` on a bad index, but it is slower.

### Key Differences from std::vector

- The size is part of the type: `std::array<int, 3>` and `std::array<int, 4>` are completely different types.
- No `push_back`, `pop_back`, or `resize` -- the element count is fixed forever.
- Allocated on the stack, so no heap allocation cost.
- Supports `constexpr` -- an entire `std::array` can be created and used at compile time.

### When to Prefer std::array

Use `std::array` whenever the element count is a compile-time constant: small lookup tables, fixed-size coordinate tuples, RGB color values, or configuration parameters that never change. Use `std::vector` when the size is determined at runtime or the collection needs to grow.
