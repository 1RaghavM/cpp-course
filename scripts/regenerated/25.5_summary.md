Every function call in your program must resolve to a specific function. The process of matching a function call to a function definition is called **binding**. C++ uses two distinct binding strategies, and understanding the difference is essential for working with virtual functions.

**Early binding** (also called static binding or compile-time binding) means the compiler determines which function to call when the program is compiled. The compiler looks at the declared type of the variable and picks the matching function directly. Non-virtual function calls, direct function calls, and overloaded function calls are all resolved this way. Early binding is fast because the compiler inserts a direct jump to the function's address -- there is no runtime decision.

**Late binding** (also called dynamic binding or runtime binding) means the function to call is determined while the program is running. When you call a `virtual` function through a base-class pointer or reference, the compiler cannot know at compile time which derived type the pointer actually points to. Instead, it generates code that looks up the correct function at runtime. This lookup involves an indirect function call -- the program reads a function address from a hidden table and then jumps to it.

```cpp
#include <iostream>
class Base {
public:
    void earlyBound() const { std::cout << "Base early\n"; }
    virtual void lateBound() const { std::cout << "Base late\n"; }
};
class Derived : public Base {
public:
    void earlyBound() const { std::cout << "Derived early\n"; }
    void lateBound() const override { std::cout << "Derived late\n"; }
};
```

Given `Derived d{}; Base& ref{d};` -- calling `ref.earlyBound()` prints `Base early` because the call is resolved at compile time using the reference's declared type (`Base`). Calling `ref.lateBound()` prints `Derived late` because `lateBound` is virtual, so the call is resolved at runtime using the object's actual type (`Derived`).

The performance cost of late binding is small -- one extra pointer dereference per call -- but it is nonzero. This is why C++ does not make every function virtual by default: you only pay for the indirection when you need runtime polymorphism. Non-virtual functions are the default because early binding is cheaper and sufficient when you do not need derived classes to override behavior through base pointers.
