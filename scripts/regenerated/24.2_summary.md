Now that you understand why inheritance exists (lesson 24.1), let us look at how to write it in C++. You create a derived class by placing a colon after its name, followed by the keyword `public` and the base class name: `class Derived : public Base`. This tells the compiler that `Derived` inherits every non-private member variable and member function that `Base` defines.

Here is a minimal example:

```cpp
#include <iostream>
#include <string>

class Animal {
    std::string m_name;
    int m_legs{};
public:
    Animal(std::string_view name, int legs)
        : m_name{name}, m_legs{legs} {}
    std::string_view getName() const { return m_name; }
    int getLegs() const { return m_legs; }
};

class Dog : public Animal {
public:
    Dog(std::string_view name)
        : Animal{name, 4} {}
    void bark() const { std::cout << "Woof!\n"; }
};
```

`Dog` publicly inherits from `Animal`. That means every `Dog` object contains the `m_name` and `m_legs` members defined by `Animal`, and the public functions `getName()` and `getLegs()` are callable through a `Dog` object, even though `Dog` never declared them. `Dog` also adds its own member function `bark()`, which only `Dog` (and its future descendants) can use. A plain `Animal` object has no `bark()` function.

Notice that `Dog`'s constructor calls `Animal`'s constructor in its member initializer list: `Animal{name, 4}`. This is necessary because a derived class does **not** inherit the base class's constructors. The derived constructor must explicitly delegate to a base constructor so that the base portion of the object is properly initialized. Destructors and copy/move assignment operators are likewise not inherited.

To summarize what a derived class inherits and what it does not:

- **Inherited:** public (and protected) member variables and member functions of the base class.
- **Not inherited:** constructors, destructors, and assignment operators. The derived class must provide its own (or the compiler generates defaults).

Because `Dog` inherits `Animal`'s public interface, you can call `getName()` and `getLegs()` on a `Dog` exactly as you would on an `Animal`. The derived class reuses the base class's implementation without duplicating code -- this is the central benefit of inheritance.
